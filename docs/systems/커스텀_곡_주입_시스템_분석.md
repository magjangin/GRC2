# 커스텀 곡 주입 시스템 분석

## 개요

이 문서는 게임의 곡 목록에 커스텀 차트를 주입하는 시스템의 상세한 동작 원리를 설명합니다.

## 시스템 목적

게임의 원본 곡 목록에 커스텀 BMS 차트를 추가하여, 곡 선택 화면에서 커스텀 차트를 선택하고 플레이할 수 있도록 합니다.

## 전체 흐름

```
1. 게임 시작 → 모드 초기화
   ↓
2. hwa 폴더 스캔 (AlbumManager.ScanAlbums)
   ├─ 각 앨범 폴더 스캔
   ├─ BMS, 이미지, BGA, BGM 파일 매핑
   └─ txt 파일 파싱 (SongInfo)
   ↓
3. 곡 목록 초기화 (initializeAllItemByCrrentMusicData)
   ├─ 원본 곡 목록 확인
   ├─ 템플릿 곡 선택
   ├─ 커스텀 MusicID 생성
   ├─ MusicSelectScrollItemData 복제
   └─ mCellHaviableMusicDataList에 추가
   ↓
4. 곡 선택 화면 표시 (커스텀 곡 포함)
   ↓
5. 노트/BGA/BGM 주입 (플레이 시)
   └─ NoteArrayHooks, BgmBgaInjector 등은 **CustomAssetManager.ShouldInjectCustomContent()**로 제어
   └─ 주입 금지 씬(SoundPlayerScene, MoviePlayer_MovieSelect)이거나 커스텀 차트 미선택 시 주입 안 함
```

## 1. 앨범 스캔 단계

### 1.1 폴더 구조

```
hwa/
├── Album1/
│   ├── song1.bms
│   ├── cover.jpg
│   ├── bgm.mp3
│   ├── bga.mp4
│   └── info.txt
├── Album2/
│   └── ...
└── (루트 파일들)
```

### 1.2 스캔 로직

**위치**: `AlbumManager.ScanAlbums(string hwaFolderPath)`

**동작**:
1. `hwa` 폴더 내 모든 하위 폴더 검색
2. 각 폴더를 앨범으로 인식
3. `hwa` 폴더 자체도 앨범으로 간주 (루트 앨범)

**파일 매핑**:
- BMS 파일: `*.bms`, `*.bme`, `*.bml`
- 이미지 파일: `*.jpg`, `*.png`, `*.jpeg`
- BGA 파일: `*.mp4`
- BGM 파일: `*.mp3`, `*.wav`, `*.ogg`
- 곡 정보 파일: `*.txt`

**AlbumInfo 구조**:
```csharp
public class AlbumInfo
{
    public string AlbumFolderPath { get; set; }
    public string AlbumName { get; set; }
    public List<string> BmsFiles { get; set; }
    public List<string> ImageFiles { get; set; }
    public List<string> BgaFiles { get; set; }
    public List<string> BgmFiles { get; set; }
    public string TxtFile { get; set; }
    public SongInfo SongInfo { get; set; }
}
```

## 2. 곡 목록 초기화 단계

### 2.1 후킹 지점

**위치**: `MusicScrollViewHooks.InitializeAllItemByCrrentMusicDataPrefix (구현: MusicScrollViewHooks.ListLogging.cs)`

**후킹 메서드**: `cMusicSelectScrollView.initializeAllItemByCrrentMusicData(bool isSceneFirst)`

**호출 시점**: 곡 선택 화면이 처음 로드될 때

### 2.2 템플릿 곡 선택

**목적**: 원본 곡 중 하나를 템플릿으로 사용하여 커스텀 곡 생성

**선택 기준**:
- `mCellHaviableMusicDataList`의 첫 번째 곡 사용
- 또는 특정 MusicID (예: `FIRST`) 사용

**템플릿 곡의 역할**:
- `MusicSelectScrollItemData` 구조 복제
- `MusicSelectData` 필드 값 복사
- 난이도 정보 복사

### 2.3 커스텀 MusicID 생성

**위치**: `MusicScrollViewHooks.GenerateCustomMusicID (구현: MusicScrollViewHooks.Injection.cs)`

**생성 규칙**:

#### enum 타입인 경우
```csharp
// 54-511 범위 사용 (458개 앨범)
int customValue = 54 + albumIndex;

// 511 초과 시 10000 이상 사용
if (customValue > 511)
{
    customValue = 10000 + (albumIndex - 458);
}
```

#### 정수 타입인 경우
```csharp
// 직접 정수 값 사용
int customValue = 54 + albumIndex;
```

**충돌 방지**:
- 기존 곡의 MusicID와 충돌하지 않는 범위 사용
- 54-511 범위는 게임의 원본 곡과 충돌하지 않음
- 10000 이상은 확장 범위

**MusicID 매핑**:
```csharp
// AlbumManager에 MusicID-앨범 매핑 등록
AlbumManager.RegisterMusicIdToAlbum(customMusicId, albumInfo);
```

## 3. 커스텀 곡 생성 단계

### 3.1 MusicSelectScrollItemData 복제

**위치**: `MusicScrollViewHooks.InjectCustomMusicToCellList (구현: MusicScrollViewHooks.Injection.cs)`

**복제 방법**:
```csharp
// MemberwiseClone 사용
object clonedItem = Activator.CreateInstance(itemType);
// 필드 복사
foreach (var field in itemType.GetFields(...))
{
    field.SetValue(clonedItem, field.GetValue(templateItem));
}
```

**복제 대상 필드**:
- `mIndex`: 인덱스 (새로운 값 할당)
- `mMusicSelectData`: MusicSelectData 객체 (복제 및 수정)

### 3.2 MusicSelectData 수정

**수정 항목**:
- `musicID`: 커스텀 MusicID로 변경
- `songTitle`: 앨범 이름 또는 txt 파일의 제목
- 기타 필드: 템플릿 곡의 값 유지

**구현**:
```csharp
// MusicSelectData 복제
object clonedMusicSelectData = MemberwiseClone(musicSelectData);

// musicID 변경
musicIdField.SetValue(clonedMusicSelectData, customMusicId);

// songTitle 변경 (선택적)
if (!string.IsNullOrEmpty(albumName))
{
    songTitleField?.SetValue(clonedMusicSelectData, albumName);
}
```

### 3.3 원본 제목 등록

**목적**: TextPatch에서 원본 제목을 커스텀 제목으로 교체하기 위해 등록

**등록 방법**:
```csharp
// 템플릿 곡의 원본 제목을 등록
string originalTitle = templateSongTitle;
AlbumManager.RegisterOriginalTitle(customMusicId, originalTitle) (구현: AlbumManager.Mappings.cs);
```

**사용 예시**:
- 템플릿 곡 제목: "푸른 경계"
- 커스텀 곡 제목: "summertime (Arrange ver.)"
- 등록: `54 -> "푸른 경계"`
- TextPatch에서 "푸른 경계"를 "summertime (Arrange ver.)"로 교체

## 4. 곡 목록에 추가

### 4.1 mCellHaviableMusicDataList에 추가

**위치**: `MusicScrollViewHooks.InjectCustomMusicToCellList (구현: MusicScrollViewHooks.Injection.cs)`

**추가 방법**:
```csharp
// IList에 직접 추가
if (cellList is IList list)
{
    list.Add(clonedItem);
}
```

**순서**:
- 원본 곡 목록 뒤에 추가
- 또는 특정 위치에 삽입 가능

### 4.2 인덱스 관리

**mIndex 필드**:
- 각 곡의 고유 인덱스
- 커스텀 곡은 새로운 인덱스 할당

**구현**:
```csharp
int newIndex = list.Count; // 다음 인덱스
indexField.SetValue(clonedItem, newIndex);
```

## 5. 아티스트 ID 기반 처리

### 5.1 아티스트 ID 확인

**위치**: `MusicScrollViewHooks.InjectCustomMusicToCellList (구현: MusicScrollViewHooks.Injection.cs)`

**확인 순서**:
1. `SongInfo.Character` (캐릭터 필드)
2. `SongInfo.Artist` (아티스트 필드)

**사용 목적**:
- 아티스트 ID별 첫 곡 정보 등록
- 곡 시작 전 MusicID 변경에 사용

### 5.2 첫 곡 정보 등록

**조건**:
- 아티스트 ID가 이미 등록되어 있지 않을 때만 등록
- 첫 곡만 유지 (나중 곡은 무시)

**등록**:
```csharp
string artistId = songInfo.Character ?? songInfo.Artist;
if (!string.IsNullOrEmpty(artistId))
{
    AlbumManager.RegisterArtistFirstSong(artistId, customMusicId, albumName) (구현: AlbumManager.Mappings.cs);
}
```

## 7. 주요 메서드

### MusicScrollViewHooks

**주요 메서드**:
- `InitializeAllItemByCrrentMusicDataPrefix()`: 곡 목록 초기화 후킹
- `InjectCustomMusicToCellList()`: 커스텀 곡 주입 로직
- `GenerateCustomMusicID()`: 커스텀 MusicID 생성
- `PrintMusicList()`: 곡 목록 출력 (디버깅용)

### AlbumManager

**주요 메서드**:
- `ScanAlbums(string hwaFolderPath)`: 앨범 폴더 스캔
- `SelectAlbum()`: 앨범 선택
- `SelectAlbumByMusicID()`: MusicID로 앨범 선택
- `RegisterMusicIdToAlbum()`: MusicID-앨범 매핑 등록
- `GetAllAlbums()`: 모든 앨범 정보 반환

## 8. 주의사항 및 제한사항

### 8.1 MusicID 범위

- 54-511 범위: 최대 458개 앨범
- 10000 이상: 확장 범위 (무제한)
- 기존 곡과 충돌하지 않도록 주의

### 8.2 템플릿 곡 의존성

- 템플릿 곡의 구조에 의존
- 게임 업데이트 시 구조 변경 가능성
- Reflection을 통한 동적 접근으로 대응

### 8.3 메모리 사용

- 모든 앨범의 파일 정보를 메모리에 보관
- 대량의 앨범이 있을 경우 메모리 사용량 증가

## 9. 디버깅 팁

### 9.1 로그 확인

**앨범 스캔**:
```
[AlbumManager] 앨범 등록: Album1 (1개 BMS, 1개 이미지, 1개 BGA, 1개 BGM)
```

**커스텀 곡 주입**:
```
[MusicScrollViewHooks]   커스텀 enum 값 생성: 숫자 54 (범위: 54-511, 앨범 인덱스: 1)
[MusicScrollViewHooks]   ✅ [1/2] 'summertime (Arrange ver.)' 추가 완료
```

**곡 목록 출력**:
```
[MusicScrollViewHooks]   📋 곡 목록:
[MusicScrollViewHooks]     [00] Index: 0, ID: FIRST, 제목: '푸른 경계'
[MusicScrollViewHooks]     [01] Index: 1, ID: 54, 제목: 'summertime (Arrange ver.)'
```

### 9.2 문제 해결

**문제**: 커스텀 곡이 목록에 나타나지 않음
- **확인**: `hwa` 폴더에 앨범 폴더가 있는지 확인
- **확인**: BMS 파일이 있는지 확인
- **확인**: 로그에서 "추가 완료" 메시지 확인

**문제**: MusicID 충돌
- **확인**: 커스텀 MusicID 범위 확인 (54-511, 10000 이상)
- **확인**: 기존 곡의 MusicID와 충돌하지 않는지 확인

**문제**: 템플릿 곡 복제 실패
- **확인**: `mCellHaviableMusicDataList`가 비어있지 않은지 확인
- **확인**: Reflection으로 필드 접근이 가능한지 확인

## 기술 스택 및 참조

### 어셈블리 어트리뷰트

프로젝트의 진입점은 다음 어트리뷰트로 정의됩니다:

```csharp
[assembly: MelonInfo(typeof(GRC2.Core.SceneDetector), "GUNVOLT RECORDS Cychronicle", "1.0.0", "")]
[assembly: MelonGame("INTI CREATES", "GUNVOLT RECORDS Cychronicle")]
```

**위치**: `GRC2/Core/Scene/SceneDetector.cs`

### 참조 DLL

이 시스템은 다음 DLL들을 참조합니다:

- **MelonLoader.dll**: MelonLoader 모딩 프레임워크
- **0Harmony.dll**: Harmony 라이브러리 (런타임 메서드 패칭)
- **UnityEngine.CoreModule.dll**: Unity 핵심 모듈
- **UnityEngine.UI.dll**: Unity UI 시스템
- **UnityEngine.AudioModule.dll**: Unity 오디오 시스템
- **UnityEngine.VideoModule.dll**: Unity 비디오 재생 모듈
- **UnityEngine.UnityWebRequestModule.dll**: Unity 웹 요청 모듈
- **UnityEngine.UnityWebRequestAudioModule.dll**: Unity 오디오 웹 요청 모듈
- **UnityEngine.ImageConversionModule.dll**: Unity 이미지 변환 모듈
- 기타 Unity 모듈들 (AnimationModule, PhysicsModule, ParticleSystemModule, UIModule, TextMeshPro)

### Harmony 패치 방식

이 시스템은 Harmony를 통한 동적 패치 방식을 사용합니다:

- `HarmonyMethod`를 사용하여 런타임에 메서드를 패치
- `harmonyInstance.Patch()` 메서드를 통해 Prefix/Postfix 패치 적용
- 어트리뷰트 기반 패치(`[HarmonyPrefix]`, `[HarmonyPostfix]`)는 사용하지 않음

**주요 패치 클래스**:
- `MusicScrollViewHooks`: 곡 목록 주입 후킹 (`initializeAllItemByCrrentMusicData` 등)
- `PatchApplier`: 패치 적용기 (각 Patcher 클래스 초기화)
