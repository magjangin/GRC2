# 앨범 관리 시스템 분석

## 개요

이 문서는 커스텀 차트의 앨범을 관리하는 `AlbumManager` 시스템의 상세한 동작 원리를 설명합니다.

## 시스템 목적

`hwa` 폴더 내의 여러 앨범 폴더를 스캔하고, 각 앨범의 파일(BMS, 이미지, BGA, BGM, txt)을 관리하며, MusicID와 앨범을 매핑하여 곡 선택 시 자동으로 해당 앨범을 선택할 수 있도록 합니다.

## 전체 구조

```
AlbumManager (정적 클래스)
├── _albums: Dictionary<string, AlbumInfo>          // 앨범 목록
├── _currentAlbum: AlbumInfo                        // 현재 선택된 앨범
├── _musicIdToAlbumMap: Dictionary<object, AlbumInfo>  // MusicID-앨범 매핑
├── _artistIdToFirstSong: Dictionary<string, (object, string)>  // 아티스트 ID별 첫 곡
└── _currentArtistId: string                        // 현재 아티스트 ID
```

## 1. 앨범 스캔

### 1.1 폴더 구조 인식

**위치**: `AlbumManager.ScanAlbums(string hwaFolderPath)`

**인식 규칙**:
- `hwa` 폴더 내의 모든 하위 폴더를 앨범으로 인식
- `hwa` 폴더 자체도 앨범으로 간주 (루트 앨범, 키: "root")

**예시**:
```
hwa/
├── Album1/          → 앨범 키: "Album1"
├── Album2/          → 앨범 키: "Album2"
└── (루트 파일들)    → 앨범 키: "root"
```

### 1.2 파일 스캔

**위치**: `AlbumManager.ScanAlbumFolder(string albumFolderPath)`

**스캔 대상 파일**:
- BMS 파일: `*.bms`, `*.bme`, `*.bml`
- 이미지 파일: `*.jpg`, `*.png`, `*.jpeg`
- BGA 파일: `*.mp4`
- BGM 파일: `*.mp3`, `*.wav`, `*.ogg`
- 곡 정보 파일: `*.txt` (단일 파일만)

**파일 매핑**:
```csharp
public class AlbumInfo
{
    public string AlbumFolderPath { get; set; }      // 앨범 폴더 경로
    public string AlbumName { get; set; }            // 앨범 이름 (폴더명)
    public List<string> BmsFiles { get; set; }       // BMS 파일 목록
    public List<string> ImageFiles { get; set; }    // 이미지 파일 목록
    public List<string> BgaFiles { get; set; }      // BGA 파일 목록
    public List<string> BgmFiles { get; set; }      // BGM 파일 목록
    public string TxtFile { get; set; }              // txt 파일 (단일)
    public SongInfo SongInfo { get; set; }           // 파싱된 곡 정보
}
```

### 1.3 txt 파일 파싱

**위치**: `AlbumManager.ScanAlbumFolder` → `SongInfoParser.ParseTxtFile`

**파싱 필드**:
- 곡 제목: "곡 제목 : " 또는 "title="
- 아티스트: "아티스트 : " 또는 "artist="
- 캐릭터: "캐릭터 : " 또는 "character="
- 난이도: "난이도 : " 또는 "difficulty="
- 난이도 숫자: "easy = 1", "normal = 2" 등

**SongInfo 구조**:
```csharp
public class SongInfo
{
    public string Title { get; set; }                // 곡 제목
    public string Artist { get; set; }              // 아티스트
    public string Character { get; set; }            // 캐릭터 (아티스트 ID)
    public List<string> Difficulties { get; set; }  // 난이도 목록
    public Dictionary<string, int> DifficultyNumbers { get; set; }  // 난이도별 숫자
}
```

## 2. 앨범 선택

### 2.1 선택 방법

**방법 1: 폴더 경로로 선택**
```csharp
AlbumManager.SelectAlbum(string albumFolderPath)
```

**방법 2: MusicID로 선택**
```csharp
AlbumManager.SelectAlbumByMusicID(object musicId) (구현: AlbumManager.Mappings.cs)
```

**방법 3: 곡 정보로 선택**
```csharp
AlbumManager.SelectAlbumBySongInfo(SongInfo songInfo)
```

### 2.2 선택 우선순위 (곡 정보로 선택 시)

1. **곡 제목 매칭**: `songInfo.Title`과 `album.SongInfo.Title` 비교
2. **아티스트 매칭**: 곡 제목이 없으면 `songInfo.Artist`와 `album.SongInfo.Artist` 비교
3. **기본 앨범**: 매칭되는 앨범이 없으면 첫 번째 앨범 선택

### 2.3 현재 앨범 정보 조회

**주요 메서드**:
- `GetCurrentAlbum()`: 현재 앨범 정보 반환
- `GetCurrentBmsFile()`: 현재 앨범의 첫 번째 BMS 파일
- `GetCurrentImageFile()`: 현재 앨범의 첫 번째 이미지 파일
- `GetCurrentBgaFile()`: 현재 앨범의 첫 번째 BGA 파일
- `GetCurrentBgmFile()`: 현재 앨범의 BGM 파일 (OGG 우선)
- `GetCurrentSongInfo()`: 현재 앨범의 곡 정보

## 3. MusicID-앨범 매핑

### 3.1 매핑 등록

**위치**: `AlbumManager.RegisterMusicIdToAlbum(object musicId, AlbumInfo album)`

**목적**: 커스텀 곡의 MusicID와 앨범을 연결하여, 곡 선택 시 자동으로 해당 앨범을 선택

**등록 시점**:
- 커스텀 곡 주입 시 (`MusicScrollViewHooks.InjectCustomMusicToCellList (구현: MusicScrollViewHooks.Injection.cs)`)
- 각 앨범마다 고유한 커스텀 MusicID 생성 후 등록

### 3.2 매핑 조회

**위치**: `AlbumManager.SelectAlbumByMusicID(object musicId) (구현: AlbumManager.Mappings.cs)`

**동작**:
1. `_musicIdToAlbumMap`에서 MusicID로 앨범 찾기
2. 찾은 앨범을 현재 앨범으로 설정
3. 앨범 정보 로그 출력

**사용 예시**:
```csharp
// 곡 선택 시
object selectedMusicId = ...;
AlbumManager.SelectAlbumByMusicID(selectedMusicId) (구현: AlbumManager.Mappings.cs);
// 이제 GetCurrentBmsFile() 등이 선택된 앨범의 파일을 반환
```

## 4. 아티스트 ID 관리

### 4.1 첫 곡 정보 등록

**위치**: `AlbumManager.RegisterArtistFirstSong(string artistId, object musicId, string title) (구현: AlbumManager.Mappings.cs)`

**목적**: 아티스트 ID별 첫 곡의 MusicID와 제목을 저장하여, 곡 시작 전 MusicID 변경에 사용

**등록 조건**:
- 아티스트 ID가 이미 등록되어 있지 않을 때만 등록
- 첫 곡만 유지 (나중 곡은 무시)

**정규화**:
- `NormalizeArtistId()`로 아티스트 ID 정규화
- 대소문자 무시, 한글/영문 변환

### 4.2 첫 곡 정보 조회

**위치**: `AlbumManager.GetArtistFirstSong(string artistId) (구현: AlbumManager.Mappings.cs)`

**동작**:
1. 아티스트 ID 정규화
2. `_artistIdToFirstSong`에서 조회
3. `(musicId, title)` 튜플 반환

### 4.3 현재 아티스트 ID 관리

**저장**: `AlbumManager.SetCurrentArtistId(string artistId) (구현: AlbumManager.Mappings.cs)`
**조회**: `AlbumManager.GetCurrentArtistId() (구현: AlbumManager.Mappings.cs)`

**사용 목적**: TextPatch에서 아티스트 ID 기반 텍스트 교체에 사용

## 5. 원본 제목 관리

### 5.1 원본 제목 등록

**위치**: `AlbumManager.RegisterOriginalTitle(object musicId, string originalTitle) (구현: AlbumManager.Mappings.cs)`

**목적**: TextPatch에서 원본 제목을 커스텀 제목으로 교체하기 위해 등록

**등록 시점**:
- 커스텀 곡 주입 시 (템플릿 곡의 제목)
- 곡 시작 전 윈도우 열기 시 (아티스트의 첫 곡 제목)

### 5.2 원본 제목 조회

**위치**: `AlbumManager.GetAllOriginalTitles() (구현: AlbumManager.Mappings.cs)`

**반환**: 모든 등록된 원본 제목의 HashSet

**사용**: TextPatch에서 현재 텍스트가 원본 제목인지 확인

## 6. 데이터 흐름

### 6.1 앨범 스캔 → 선택 흐름

```
1. 게임 시작
   ↓
2. AlbumManager.ScanAlbums("hwa")
   ├─ hwa 폴더 내 모든 하위 폴더 스캔
   ├─ 각 폴더의 파일 매핑
   ├─ txt 파일 파싱 (SongInfo)
   └─ _albums에 등록
   ↓
3. 기본 앨범 선택 (첫 번째 또는 "root")
   ↓
4. 곡 선택 시 AlbumManager.SelectAlbumByMusicID(musicId) (구현: AlbumManager.Mappings.cs)
   └─ _musicIdToAlbumMap에서 앨범 찾기
   └─ _currentAlbum 설정
```

### 6.2 MusicID-앨범 매핑 흐름

```
1. 커스텀 곡 주입 시
   ├─ 커스텀 MusicID 생성 (54, 55, ...)
   ├─ AlbumManager.RegisterMusicIdToAlbum(54, album1)
   └─ AlbumManager.RegisterMusicIdToAlbum(55, album2)
   ↓
2. 곡 선택 시
   ├─ MusicID: 54
   ├─ AlbumManager.SelectAlbumByMusicID(54) (구현: AlbumManager.Mappings.cs)
   └─ album1이 현재 앨범으로 설정
```

### 6.3 아티스트 ID 기반 흐름

```
1. 곡 목록 초기화 시
   ├─ 각 곡의 아티스트 ID 확인
   ├─ 첫 곡 정보 등록
   └─ AlbumManager.RegisterArtistFirstSong("Morpho", GV3_001, "푸른 경계") (구현: AlbumManager.Mappings.cs)
   ↓
2. 커스텀 차트 선택 시
   ├─ txt 파일에서 캐릭터(아티스트 ID) 확인
   ├─ AlbumManager.GetArtistFirstSong("Morpho") (구현: AlbumManager.Mappings.cs)
   └─ 반환: (GV3_001, "푸른 경계")
```

## 7. 주요 메서드 요약

### 앨범 스캔
- `ScanAlbums(string hwaFolderPath)`: hwa 폴더 스캔
- `ScanAlbumFolder(string albumFolderPath)`: 특정 앨범 폴더 스캔

### 앨범 선택
- `SelectAlbum(string albumFolderPath)`: 폴더 경로로 선택
- `SelectAlbumByMusicID(object musicId)`: MusicID로 선택
- `SelectAlbumBySongInfo(SongInfo songInfo)`: 곡 정보로 선택

### 현재 앨범 정보 조회
- `GetCurrentAlbum()`: 현재 앨범 정보
- `GetCurrentBmsFile()`: 현재 BMS 파일
- `GetCurrentImageFile()`: 현재 이미지 파일
- `GetCurrentBgaFile()`: 현재 BGA 파일
- `GetCurrentBgmFile()`: 현재 BGM 파일
- `GetCurrentSongInfo()`: 현재 곡 정보

### MusicID-앨범 매핑
- `RegisterMusicIdToAlbum(object musicId, AlbumInfo album)`: 매핑 등록
- `GetMusicIDByAlbum(AlbumInfo album)`: 앨범으로 MusicID 찾기
- `IsCustomChartMusicID(object musicId)`: 커스텀 차트 MusicID 확인

### 아티스트 ID 관리
- `NormalizeArtistId(string artistId)`: 아티스트 ID 정규화
- `RegisterArtistFirstSong(string artistId, object musicId, string title)`: 첫 곡 정보 등록
- `GetArtistFirstSong(string artistId)`: 첫 곡 정보 조회
- `SetCurrentArtistId(string artistId)`: 현재 아티스트 ID 저장
- `GetCurrentArtistId()`: 현재 아티스트 ID 조회

### 원본 제목 관리
- `RegisterOriginalTitle(object musicId, string originalTitle)`: 원본 제목 등록
- `GetAllOriginalTitles()`: 모든 원본 제목 조회

## 8. 주의사항 및 제한사항

### 8.1 앨범 키 중복

- 동일한 폴더명이 있으면 나중 것이 덮어씀
- 폴더명을 고유하게 유지해야 함

### 8.2 txt 파일

- 각 앨범 폴더당 하나의 txt 파일만 인식
- 여러 txt 파일이 있으면 첫 번째만 사용

### 8.3 BGM 파일 우선순위

- OGG 파일이 있으면 우선 선택
- 없으면 첫 번째 BGM 파일 사용

### 8.4 메모리 사용

- 모든 앨범의 파일 경로를 메모리에 보관
- 대량의 앨범이 있을 경우 메모리 사용량 증가

## 9. 디버깅 팁

### 9.1 로그 확인

**앨범 스캔**:
```
[AlbumManager] 앨범 폴더 스캔 시작
[AlbumManager] 2개 앨범 폴더 발견
[AlbumManager] 앨범 등록: Album1 (1개 BMS, 1개 이미지, 1개 BGA, 1개 BGM)
```

**앨범 선택**:
```
[AlbumManager] MusicID로 앨범 선택: Album1 (MusicID: 54)
```

**아티스트 ID 등록**:
```
[AlbumManager] 아티스트 ID별 첫 곡 등록: Morpho -> (GV3_001, 푸른 경계)
```

### 9.2 문제 해결

**문제**: 앨범이 스캔되지 않음
- **확인**: `hwa` 폴더가 존재하는지 확인
- **확인**: 폴더 내에 BMS 파일이 있는지 확인
- **확인**: 로그에서 "앨범 등록" 메시지 확인

**문제**: 앨범 선택이 안 됨
- **확인**: MusicID-앨범 매핑이 등록되어 있는지 확인
- **확인**: `SelectAlbumByMusicID` 호출 시 로그 확인

**문제**: 파일을 찾을 수 없음
- **확인**: `GetCurrentBmsFile()` 등이 null을 반환하는지 확인
- **확인**: 앨범 폴더에 해당 파일이 있는지 확인

## 기술 스택 및 참조

### 어셈블리 어트리뷰트

프로젝트의 진입점은 다음 어트리뷰트로 정의됩니다:

```csharp
[assembly: MelonInfo(typeof(GRC2.Core.SceneDetector), "GUNVOLT RECORDS Cychronicle", "1.0.0", "")]
[assembly: MelonGame("INTI CREATES", "GUNVOLT RECORDS Cychronicle")]
```

**위치**: `GRC2/Core/Scene/SceneDetector.cs`

### 참조 DLL

이 시스템은 다음 DLL들을 참조합니다:

- **MelonLoader.dll**: MelonLoader 모딩 프레임워크
- **0Harmony.dll**: Harmony 라이브러리 (런타임 메서드 패칭)
- **UnityEngine.CoreModule.dll**: Unity 핵심 모듈
- **UnityEngine.UI.dll**: Unity UI 시스템
- **UnityEngine.AudioModule.dll**: Unity 오디오 시스템
- **UnityEngine.VideoModule.dll**: Unity 비디오 재생 모듈
- **UnityEngine.UnityWebRequestModule.dll**: Unity 웹 요청 모듈
- **UnityEngine.UnityWebRequestAudioModule.dll**: Unity 오디오 웹 요청 모듈
- **UnityEngine.ImageConversionModule.dll**: Unity 이미지 변환 모듈
- 기타 Unity 모듈들 (AnimationModule, PhysicsModule, ParticleSystemModule, UIModule, TextMeshPro)

### Harmony 패치 방식

`AlbumManager`는 Harmony 패치를 직접 사용하지 않지만, 시스템 전반에서 Harmony를 통한 동적 패치 방식을 사용합니다:

- `HarmonyMethod`를 사용하여 런타임에 메서드를 패치
- `harmonyInstance.Patch()` 메서드를 통해 Prefix/Postfix 패치 적용
- 어트리뷰트 기반 패치(`[HarmonyPrefix]`, `[HarmonyPostfix]`)는 사용하지 않음
