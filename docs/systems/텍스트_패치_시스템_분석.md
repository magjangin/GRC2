# 텍스트 패치 시스템 분석

## 개요

이 문서는 게임 화면에 표시되는 텍스트를 가로채서 커스텀 차트의 제목으로 교체하는 `TextPatch` 시스템의 상세한 동작 원리를 설명합니다.

## 시스템 목적

플레이 씬, 로딩 씬, 결과 씬에서 원본 곡의 제목이 표시될 때, 이를 커스텀 차트의 제목으로 자동 교체하여 사용자에게 올바른 곡 제목을 표시합니다.

## 전체 흐름

```
1. 커스텀 차트 선택 (coOpen)
   ↓
2. TextPatch.EnableTextReplacement(true)
   └─ 텍스트 교체 스위치 활성화
   ↓
3. 플레이 씬/로딩 씬/결과 씬 진입
   ↓
4. 게임이 텍스트를 설정할 때 (Text.text 또는 TextMeshProUGUI.text setter)
   ↓
5. TextPatch.SetTextPrefix 후킹
   ├─ 스위치 확인 (활성화되어 있는지)
   ├─ 씬 확인 (플레이/로딩/결과 씬인지)
   ├─ 원본 제목 확인 (등록된 원본 제목인지)
   ├─ 현재 앨범의 커스텀 제목 가져오기
   └─ 텍스트 교체
```

## 1. 후킹 설정

### 1.1 후킹 대상

**위치**: `PatchApplier.PatchTextTypes()` (`GRC2/Injectors/Shared/PatchApplier.cs`)

**후킹 대상 타입**:
- `UnityEngine.UI.Text`
- `TMPro.TextMeshProUGUI`
- `TMPro.TextMeshPro`

**후킹 대상 속성**:
- `text` 속성의 setter

### 1.2 후킹 방법

**Harmony Prefix 패치**:
```csharp
// Text.text setter 후킹
var textProperty = textType.GetProperty("text", BindingFlags.Public | BindingFlags.Instance);
if (textProperty != null && textProperty.SetMethod != null)
{
    harmonyInstance.Patch(
        textProperty.SetMethod,
        prefix: new HarmonyMethod(typeof(TextPatch).GetMethod("SetTextPrefix"))
    );
}
```

**특징**:
- `ref string value`를 사용하여 원본 값을 수정 가능
- Prefix이므로 원본 메서드 실행 전에 텍스트 교체

## 2. 활성화/비활성화

### 2.1 스위치 시스템

**위치**: `TextPatch._isTextReplacementEnabled`

**상태 관리**:
- `false` (기본값): 텍스트 교체 비활성화
- `true`: 텍스트 교체 활성화

**활성화 시점**:
- 커스텀 차트 선택 시 (`GameFlowHooks.CoOpenPrefix (구현: GameFlowHooks.Navigation.cs)`)
- `TextPatch.EnableTextReplacement(true)` 호출

**비활성화 시점**:
- 일반 곡 선택 시 (`GameFlowHooks.CoOpenPrefix (구현: GameFlowHooks.Navigation.cs)`)
- `TextPatch.EnableTextReplacement(false)` 호출

### 2.2 활성화 조건

**조건 1: 스위치 활성화**
```csharp
if (!_isTextReplacementEnabled)
{
    return; // 텍스트 교체하지 않음
}
```

**조건 2: 플레이/로딩/결과 씬**
```csharp
if (!IsPlayOrLoadingScene())
{
    return; // 해당 씬이 아니면 교체하지 않음
}
```

**지원 씬**:
- `FairyModeScene`: 페어리 모드 플레이 씬
- `PlayMovieScene`: 영상 재생 씬
- `RenderCutinScene`: 로딩 씬
- `RythmGameResultScene`: 결과 씬

## 3. 텍스트 교체 로직

### 3.1 원본 제목 확인

**목적**: 커스텀 차트가 사용한 원본 곡의 제목만 교체

**확인 방법**:
```csharp
var originalTitles = AlbumManager.GetAllOriginalTitles() (구현: AlbumManager.Mappings.cs);
if (!originalTitles.Contains(value))
{
    return; // 원본 제목이 아니면 교체하지 않음
}
```

**원본 제목 등록 시점**:
1. 커스텀 곡 주입 시: 템플릿 곡의 제목을 원본 제목으로 등록
2. 곡 시작 전 윈도우 열기 시: 아티스트의 첫 곡 제목을 원본 제목으로 등록

### 3.2 커스텀 제목 가져오기

**위치**: `AlbumManager.GetCurrentSongInfo()`

**가져오는 정보**:
- `SongInfo.Title`: 커스텀 차트의 제목 (txt 파일에서 파싱)

**확인 사항**:
```csharp
var currentSongInfo = AlbumManager.GetCurrentSongInfo();
if (currentSongInfo == null || string.IsNullOrWhiteSpace(currentSongInfo.Title))
{
    return; // 커스텀 제목이 없으면 교체하지 않음
}
```

### 3.3 텍스트 교체

**교체 방법**:
```csharp
// ref string value를 직접 수정
value = currentSongInfo.Title;
```

**중복 교체 방지**:
```csharp
if (value == currentSongInfo.Title)
{
    return; // 이미 커스텀 제목이면 교체하지 않음
}
```

## 4. 데이터 흐름

### 4.1 원본 제목 등록 흐름

```
1. 커스텀 곡 주입 시
   ├─ 템플릿 곡 제목: "푸른 경계"
   ├─ 커스텀 MusicID: 54
   └─ AlbumManager.RegisterOriginalTitle(54, "푸른 경계") (구현: AlbumManager.Mappings.cs)
   ↓
2. 곡 시작 전 윈도우 열기 시
   ├─ 아티스트 ID: "Morpho"
   ├─ 첫 곡 제목: "푸른 경계"
   └─ AlbumManager.RegisterOriginalTitle(GV3_001, "푸른 경계") (구현: AlbumManager.Mappings.cs)
```

### 4.2 텍스트 교체 흐름

```
1. 게임이 텍스트 설정: "푸른 경계"
   ↓
2. TextPatch.SetTextPrefix 후킹
   ├─ 원본 제목 확인: "푸른 경계" ✓
   ├─ 현재 앨범의 커스텀 제목: "summertime (Arrange ver.)"
   └─ 텍스트 교체: "푸른 경계" → "summertime (Arrange ver.)"
   ↓
3. 화면에 표시: "summertime (Arrange ver.)"
```

## 5. 씬 확인 로직

### 5.1 확인 방법

**방법 1: BgmBgaInjector 상태 확인**
```csharp
if (BgmBgaInjector.IsPlayScene())
{
    return true;
}
```

**방법 2: 씬 이름 확인**
```csharp
var currentScene = SceneManager.GetActiveScene();
string sceneName = currentScene.name;
return sceneName == "FairyModeScene" || 
       sceneName == "PlayMovieScene" || 
       sceneName == "RenderCutinScene" ||
       sceneName == "RythmGameResultScene";
```

### 5.2 씬별 동작

**플레이 씬** (`FairyModeScene`, `PlayMovieScene`):
- 게임 플레이 중 곡 제목 표시
- 커스텀 제목으로 교체

**로딩 씬** (`RenderCutinScene`):
- 게임 시작 전 로딩 화면
- 커스텀 제목으로 교체

**결과 씬** (`RythmGameResultScene`):
- 게임 종료 후 결과 화면
- 커스텀 제목으로 교체

**기타 씬**:
- 텍스트 교체하지 않음 (곡 선택 화면 등)

### 5.3 곡 목록 제목: 조회 경로와 UI setter(쓰기) 경로

런타임 로그로 확인한 동작을 문서화한다. (곡 선택 씬 예: `MusicSelectScene_Hasegawa`)

**1) UI에는 항상 `text` setter(쓰기)가 있다**

- 목록에 `"푸른 경계"` 같은 문자열이 보일 때도, 최종적으로는 `TMPro.TextMeshProUGUI` 또는 `UnityEngine.UI.Text`의 **`text` 프로퍼티 setter**가 호출된다.
- `TextPatch.SetTextPrefix`는 이 setter에 Harmony **Prefix**로 붙어 `ref string value`를 바꿀 수 있다.

**2) 곡 선택 씬에서는 TextPatch가 교체하지 않는다**

- 조건: `_isTextReplacementEnabled == true` **그리고** `IsPlayOrLoadingScene() == true`일 때만 교체 로직이 진행된다.
- 곡 선택 씬에서는 보통 `playLikeScene == false`, 커스텀 미선택 시 `replacementOn == false`이므로 **조기 return**한다.
- 그 결과 setter는 **여전히 호출되지만**, GRC2는 값을 바꾸지 않아 **게임이 넘긴 원본 제목 문자열이 그대로** UI에 쓰인다. (의도된 제한)

**3) `getMusicTitle`은 별도의 조회(반환) 경로다**

- `MusicInjector`에서 `cMusicSelectScrollViewDataGetter.getMusicTitle`에 **Postfix**(`MusicTitlePatch.GetMusicTitlePostfix`)를 붙인다.
- 여기서는 `ref string __result`로 **조회 결과**를 바꿀 수 있다.

**4) MusicTitlePatch가 목록에서 안 먹히는 대표 이유**

- `AlbumManager.IsCustomChartMusicID(key)`는 `_musicIdToAlbumMap`에 등록된 **MusicID(object)** 를 키로 본다.
- 실제 게임은 `getMusicTitle`에 `GV3_001_NAME`, `GV3_001_RUBY` 등 **로컬라이즈용 문자열 키**를 넘기는 경우가 많다. 이 키는 주입 시 등록한 커스텀 MusicID(예: 정수 `54`)와 **다르면** `IsCustomChartMusicID`가 false가 되어 **커스텀 제목 치환이 적용되지 않는다**.

**5) 한 줄 정리**

- **표시 직전**: 쓰기 경로(`text` setter)가 맞다.
- **GRC2 제목 교체**: 플레이/로딩/결과 씬에서는 주로 `TextPatch`; 곡 선택 씬에서는 setter는 도나 교체는 하지 않으며, 목록 제목을 바꾸려면 **`getMusicTitle` 쪽 키와 `MusicID` 매핑을 맞추거나** setter를 곡 선택 씬까지 허용하는 등 **별도 설계**가 필요하다.

## 6. 주요 메서드

### TextPatch

**주요 메서드**:
- `SetTextPrefix(object __instance, ref string value)`: 텍스트 setter 후킹
- `EnableTextReplacement(bool enable)`: 텍스트 교체 활성화/비활성화
- `IsPlayOrLoadingScene()`: 플레이/로딩/결과 씬 확인

### AlbumManager

**관련 메서드**:
- `GetAllOriginalTitles()`: 모든 원본 제목 조회
- `GetCurrentSongInfo()`: 현재 앨범의 곡 정보 조회
- `RegisterOriginalTitle(object musicId, string originalTitle)`: 원본 제목 등록
- `IsCustomChartMusicID(object musicID)`: `_musicIdToAlbumMap`에 키가 있는지 (MusicTitlePatch에서 사용; `getMusicTitle`의 `key`가 이 키와 같아야 치환 가능)

### MusicTitlePatch

**위치**: `Harmony/Handlers/MusicTitlePatch.cs`, `MusicInjector.DelayedPatch()`에서 `getMusicTitle` Postfix 등록

**역할**: `getMusicTitle`이 반환한 문자열을 커스텀 앨범 제목으로 바꿀 수 있다. 다만 **5.3절**처럼 `key`가 로컬라이즈용 문자열이면 `IsCustomChartMusicID`와 맞지 않을 수 있다.

## 7. 주의사항 및 제한사항

### 7.1 성능

- 모든 텍스트 setter 호출 시 후킹 실행
- 원본 제목 확인을 위해 HashSet 조회 (O(1))
- 불필요한 교체 방지를 위한 조기 반환

### 7.2 씬 제한

- 플레이/로딩/결과 씬에서만 작동
- 곡 선택 화면에서는 작동하지 않음 (원본 제목 표시)

### 7.3 원본 제목 등록

- 커스텀 곡 주입 시점에 등록되어야 함
- 등록되지 않은 원본 제목은 교체되지 않음

### 7.4 중복 교체 방지

- 이미 커스텀 제목이면 교체하지 않음
- 무한 루프 방지

## 8. 디버깅 팁

### 8.1 로그 확인

**텍스트 교체 활성화**:
```
[TextPatch] 텍스트 교체 스위치: ON
```

**텍스트 교체 실행**:
```
[TextPatch] ✅ 텍스트 교체: '푸른 경계' -> 'summertime (Arrange ver.)'
```

### 8.2 문제 해결

**문제**: 텍스트가 교체되지 않음
- **확인**: `_isTextReplacementEnabled`가 `true`인지 확인
- **확인**: 현재 씬이 플레이/로딩/결과 씬인지 확인
- **확인**: 원본 제목이 `GetAllOriginalTitles()`에 등록되어 있는지 확인
- **확인**: 현재 앨범의 커스텀 제목이 있는지 확인

**문제**: 텍스트가 계속 교체됨
- **확인**: `value == currentSongInfo.Title` 체크가 제대로 작동하는지 확인
- **확인**: 원본 제목과 커스텀 제목이 다른지 확인

**문제**: 잘못된 텍스트로 교체됨
- **확인**: `GetCurrentSongInfo()`가 올바른 앨범의 정보를 반환하는지 확인
- **확인**: `SelectAlbumByMusicID()`가 올바르게 호출되었는지 확인

**문제**: 곡 선택 화면(목록)에서 커스텀 제목이 그대로 안 바뀜
- **TextPatch**: 곡 선택 씬은 `IsPlayOrLoadingScene()` 대상이 아니므로 **교체 자체가 스킵**되는 것이 정상이다. (**5.3절**)
- **MusicTitlePatch**: `getMusicTitle`에 넘어오는 `key`가 `IsCustomChartMusicID`에 걸리는지 확인한다. 로컬라이즈 키(`GV3_001_NAME` 등)만 오면 매핑 키(MusicID)와 맞추는 추가 작업이 필요할 수 있다. (**5.3절**)

## 9. 사용 예시

### 9.1 기본 사용

```csharp
// 1. 커스텀 차트 선택 시
TextPatch.EnableTextReplacement(true);

// 2. 플레이 씬에서 게임이 "푸른 경계"를 표시하려고 할 때
// TextPatch.SetTextPrefix가 자동으로 호출되어
// "summertime (Arrange ver.)"로 교체

// 3. 일반 곡 선택 시
TextPatch.EnableTextReplacement(false);
```

### 9.2 원본 제목 등록

```csharp
// 커스텀 곡 주입 시
AlbumManager.RegisterOriginalTitle(customMusicId, "푸른 경계") (구현: AlbumManager.Mappings.cs);

// 곡 시작 전 윈도우 열기 시
AlbumManager.RegisterOriginalTitle(firstSongMusicId, "푸른 경계") (구현: AlbumManager.Mappings.cs);
```

## 기술 스택 및 참조

### 어셈블리 어트리뷰트

프로젝트의 진입점은 다음 어트리뷰트로 정의됩니다:

```csharp
[assembly: MelonInfo(typeof(GRC2.Core.SceneDetector), "GUNVOLT RECORDS Cychronicle", "1.0.0", "")]
[assembly: MelonGame("INTI CREATES", "GUNVOLT RECORDS Cychronicle")]
```

**위치**: `GRC2/Core/Scene/SceneDetector.cs`

### 참조 DLL

이 시스템은 다음 DLL들을 참조합니다:

- **MelonLoader.dll**: MelonLoader 모딩 프레임워크
- **0Harmony.dll**: Harmony 라이브러리 (런타임 메서드 패칭)
- **UnityEngine.CoreModule.dll**: Unity 핵심 모듈
- **UnityEngine.UI.dll**: Unity UI 시스템
- **Unity.TextMeshPro.dll**: TextMeshPro 텍스트 렌더링
- **UnityEngine.AudioModule.dll**: Unity 오디오 시스템
- **UnityEngine.VideoModule.dll**: Unity 비디오 재생 모듈
- **UnityEngine.UnityWebRequestModule.dll**: Unity 웹 요청 모듈
- **UnityEngine.UnityWebRequestAudioModule.dll**: Unity 오디오 웹 요청 모듈
- **UnityEngine.ImageConversionModule.dll**: Unity 이미지 변환 모듈
- 기타 Unity 모듈들 (AnimationModule, PhysicsModule, ParticleSystemModule, UIModule)

### Harmony 패치 방식

이 시스템은 Harmony를 통한 동적 패치 방식을 사용합니다:

- `HarmonyMethod`를 사용하여 런타임에 메서드를 패치
- `harmonyInstance.Patch()` 메서드를 통해 Prefix/Postfix 패치 적용
- 어트리뷰트 기반 패치(`[HarmonyPrefix]`, `[HarmonyPostfix]`)는 사용하지 않음

**주요 패치 클래스**:
- `TextPatcher`: 텍스트 설정 관련 타입 패치 (`UnityEngine.UI.Text`, `TMPro.TextMeshProUGUI`, `TMPro.TextMeshPro`)
- `PatchApplier`: 패치 적용기 (각 Patcher 클래스 초기화)
