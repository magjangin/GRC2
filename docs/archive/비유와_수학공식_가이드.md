# 🎵 GRC2 모드: 비유와 핵심 수학 공식 가이드

## 📚 전체 시스템 비유: 콘서트 공연 제작 시스템

이 모드는 **음악 게임을 위한 커스텀 콘서트 제작 시스템**입니다. 마치 무대 뒤에서 모든 것을 조율하는 프로듀서처럼 작동합니다.

## 🔢 핵심 수학 공식 (중학생 수준)

### 1. 시간 변환 공식: Tick → Time → Sample

**비유**: 악보의 마디(Tick)를 실제 시간(초)으로 바꾸고, 다시 오디오 샘플로 변환하는 과정

#### 공식 1-1: Tick을 Time(초)으로 변환 (BPM 변화 없을 때 - 단순 버전)

**⚠️ 주의**: 이 공식은 **BPM이 한 번도 바뀌지 않을 때만** 사용 가능합니다!

```
Time(초) = Tick × 4 × (60 / BPM)
```

**간단한 버전**:
```
Time(초) = Tick × (240 / BPM)
```

**설명**:
- `Tick`: BMS 파일의 마디 단위 위치 (예: 1.5 = 1마디 반)
- `4`: 1마디 = 4박자
- `60 / BPM`: 1박자당 초 (BPM = beats per minute)
- `240 = 4 × 60`: 상수로 합친 값

**예시**:
- BPM = 120, Tick = 2.0
- Time = 2.0 × (240 / 120) = 2.0 × 2 = **4.0초**

---

#### 공식 1-2: Tick을 Time(초)으로 변환 (BPM 변화 있을 때 - 복잡 버전)

**⚠️ 중요**: BPM이 중간에 바뀌면 **구간별로 누적 계산**해야 합니다!

**알고리즘**:
```
1. BPM 변화 지점들을 Tick 순서로 정렬
2. 각 구간별로 시간 계산 후 누적
3. 마지막 구간까지 계산
```

**수식**:
```
전체 시간 = Σ(구간 길이 × 4 × 해당 구간의 Freq)
```

여기서:
- `구간 길이 = 다음 BPM 변화 Tick - 현재 Tick`
- `Freq = 60 / BPM` (해당 구간의 주파수)

**단계별 계산**:

```
초기값:
  time = 0
  lastTick = 0
  lastFreq = 60 / baseBPM

각 BPM 변화 지점마다:
  if (변화 Tick > 현재 노트 Tick) break
  
  // 이전 구간 계산
  if (변화 Tick > lastTick):
    offset = 변화 Tick - lastTick
    time += offset × 4 × lastFreq
    lastTick = 변화 Tick
  
  lastFreq = 60 / 새로운 BPM

// 마지막 구간 (현재 노트 Tick까지)
if (현재 노트 Tick > lastTick):
  offset = 현재 노트 Tick - lastTick
  time += offset × 4 × lastFreq
```

**예시** (복잡한 케이스):

**상황**:
- 기본 BPM = 120
- Tick 2.0에서 BPM → 150으로 변경
- Tick 5.0에서 BPM → 100으로 변경
- **계산할 노트**: Tick = 6.0

**계산 과정**:

1. **구간 1** (0 ~ 2.0마디, BPM = 120):
   - offset = 2.0 - 0 = 2.0
   - time1 = 2.0 × 4 × (60/120) = 2.0 × 4 × 0.5 = **4.0초**
   - 누적: time = 4.0초

2. **구간 2** (2.0 ~ 5.0마디, BPM = 150):
   - offset = 5.0 - 2.0 = 3.0
   - time2 = 3.0 × 4 × (60/150) = 3.0 × 4 × 0.4 = **4.8초**
   - 누적: time = 4.0 + 4.8 = **8.8초**

3. **구간 3** (5.0 ~ 6.0마디, BPM = 100):
   - offset = 6.0 - 5.0 = 1.0
   - time3 = 1.0 × 4 × (60/100) = 1.0 × 4 × 0.6 = **2.4초**
   - 누적: time = 8.8 + 2.4 = **11.2초**

**최종 답**: Tick 6.0 = **11.2초**

**비교**:
- 단순 공식 사용 시 (BPM = 120 가정): 6.0 × (240/120) = **12.0초** ❌ (틀림!)
- 구간별 계산: **11.2초** ✅ (정확!)

**핵심**: BPM이 바뀌면 단순 공식은 **절대 사용하면 안 됩니다!** 구간별로 누적 계산해야 합니다!

#### 공식 2: Time(초)을 Sample(샘플)로 변환

```
perfectSample = Time(초) × 48000
```

**설명**:
- 게임 오디오는 **48000Hz** 샘플레이트 사용
- 1초 = 48000 샘플

**예시**:
- Time = 4.0초
- perfectSample = 4.0 × 48000 = **192,000 샘플**

#### 공식 3: Sample을 Time(초)로 역변환

```
Time(초) = Sample / 48000
```

**예시**:
- Sample = 192,000
- Time = 192,000 / 48000 = **4.0초**

### 2. BPM과 시간의 관계

**비유**: 음악의 속도(BPM)가 빠를수록 같은 마디를 더 빠르게 연주

#### 공식: BPM과 1박자 시간

```
1박자 시간(초) = 60 / BPM
```

**설명**:
- BPM = 120이면 1분에 120박자
- 1박자 = 60초 / 120 = **0.5초**

**예시**:
- BPM = 100 → 1박자 = 60 / 100 = **0.6초**
- BPM = 150 → 1박자 = 60 / 150 = **0.4초**

#### 공식: BPM 변화 시 시간 계산

BPM이 중간에 바뀌면 구간별로 계산:

```
전체 시간 = 구간1 시간 + 구간2 시간 + 구간3 시간 + ...
```

**예시**:
- 0~2마디: BPM = 120
- 2~4마디: BPM = 150
- 구간1: 2 × (240 / 120) = 4.0초
- 구간2: 2 × (240 / 150) = 3.2초
- **전체 시간 = 4.0 + 3.2 = 7.2초**

---

### 3. 홀드/페어리 노트 길이 계산

**비유**: 홀드 노트는 시작과 끝이 쌍을 이루어야 함 (손가락을 누르고 있는 시간)

#### 공식: 홀드 노트 Duration

```
Duration(초) = 끝 노트 Time - 시작 노트 Time
```

**예시**:
- 시작 노트: Time = 2.0초
- 끝 노트: Time = 5.0초
- Duration = 5.0 - 2.0 = **3.0초** (3초 동안 누르고 있어야 함)

#### 공식: 홀드 노트 Sample 차이

```
Duration(Sample) = 끝 노트 perfectSample - 시작 노트 perfectSample
```

**예시**:
- 시작: perfectSample = 96,000 (2.0초)
- 끝: perfectSample = 240,000 (5.0초)
- Duration = 240,000 - 96,000 = **144,000 샘플** (= 3.0초)

### 4. 게임 종료 시간 조정 공식

**비유**: 원래 게임은 120초 후에 끝나지만, 우리 음악은 180초이므로 게임이 끝나지 않도록 조정

#### 공식: 4개 필드 조정

게임이 정확히 종료되려면 **4개 필드를 모두 조정**해야 합니다:

```
1. musicFadeOutEndSample = BGM 길이(초) × 48000
2. musicFadeOutStartSample = (BGM 길이 - 1초) × 48000
3. screenFadeOutStartSample = (BGM 길이 - 1.5초) × 48000
4. screenFadeOutEndSample = BGM 길이(초) × 48000
```

**예시** (BGM 길이 = 158.016초):
- `musicFadeOutEndSample` = 158.016 × 48000 = **7,584,768 샘플**
- `musicFadeOutStartSample` = 157.016 × 48000 = **7,536,768 샘플**
- `screenFadeOutStartSample` = 156.516 × 48000 = **7,512,768 샘플**
- `screenFadeOutEndSample` = 158.016 × 48000 = **7,584,768 샘플**

**핵심**: 하나라도 빠지면 게임이 원래 시간(120초)에 끝납니다!

### 5. BGA/BGM 동기화 공식

**비유**: 배경 영상(BGA)과 음악(BGM)이 서로 맞춰서 재생되어야 함

#### 공식: BGA 시간 동기화

```
BGA 시간 = BGM 시간 % 비디오 길이
```

**설명**:
- `%`는 나머지 연산 (모듈로 연산)
- 비디오가 루프(반복) 재생되므로, 비디오 길이로 나눈 나머지를 사용

**예시**:
- BGM 시간 = 65.0초
- 비디오 길이 = 30.0초
- BGA 시간 = 65.0 % 30.0 = **5.0초** (비디오의 5초 지점부터 재생)

#### 공식: 동기화 오차 확인

```
시간 차이 = |BGA 실제 시간 - BGA 예상 시간|
```

**설명**:
- `| |`는 절댓값 (음수 제거)
- 시간 차이가 0.1초 이상이면 재동기화

**예시**:
- BGA 실제 시간 = 5.2초
- BGA 예상 시간 = 5.0초
- 시간 차이 = |5.2 - 5.0| = **0.2초** → 재동기화 필요!

### 6. Dictionary 기반 검색 (O(1) 검색)

**비유**: 전화번호부처럼 바로 찾기 vs. 처음부터 끝까지 읽기

#### 공식: 검색 시간 비교

**선형 검색 (O(n))**:
```
검색 시간 = 노트 개수 × 평균 비교 시간
```

**예시**:
- 노트 1000개, 비교 시간 0.001초
- 검색 시간 = 1000 × 0.001 = **1.0초**

**Dictionary 검색 (O(1))**:
```
검색 시간 = 상수 시간 (약 0.0001초)
```

**예시**:
- 노트 1000개여도 검색 시간 = **0.0001초** (거의 즉시!)

**핵심**: 노트가 많을수록 Dictionary가 훨씬 빠릅니다!

---

### 7. 앨범 관리: MusicID 매핑

**비유**: 각 앨범마다 고유한 번호를 부여하여 곡 선택 시 자동으로 찾기

#### 공식: 커스텀 MusicID 생성

```
커스텀 MusicID = 54 + 앨범 인덱스
```

**설명**:
- 54-511 범위: 최대 458개 앨범 (511 - 54 + 1 = 458)
- 511 초과 시: 10000 + (앨범 인덱스 - 458)

**예시**:
- 앨범 1 → MusicID = 54 + 1 = **55**
- 앨범 2 → MusicID = 54 + 2 = **56**
- 앨범 500 → MusicID = 10000 + (500 - 458) = **10042**

## 🎬 전체 흐름 비유 (수학 공식 포함)

### 1단계: 악보 읽기 (BMS 파싱)

**비유**: 복잡한 악보를 읽어서 게임이 이해할 수 있는 형식으로 번역

```
BMS 파일 (악보)
   ↓ [BmsParser]
BmsNote 리스트 (번역된 악보)
   ├─ BPM 변화 없으면: Time = Tick × (240 / BPM)  ← 공식 1-1 (단순)
   ├─ BPM 변화 있으면: Time = Σ(구간별 계산)  ← 공식 1-2 (복잡!)
   └─ Duration = 끝 Time - 시작 Time  ← 공식 3
```

**⚠️ 중요**: BPM이 바뀌면 단순 공식은 절대 사용하면 안 됩니다!

### 2단계: 게임 형식으로 변환

**비유**: 번역된 악보를 게임의 "연주 지시서" 형식으로 변환

```
BmsNote 리스트
   ↓ [BmsNoteConverter]
NoteCreateData 배열
   ├─ perfectSample = Time × 48000  ← 공식 2
   └─ connectNodeDataArray (홀드/페어리 끝 노트)
```

### 3단계: 무대 뒤에서 조정 (Harmony 후킹)

**비유**: 게임이 노트를 생성하려고 할 때, 무대 뒤에서 조용히 우리의 노트로 교체

```
게임: "노트를 생성하겠어!" (createAllNote 호출)
   ↓
🪝 NoteArrayHooks (가로채기!)
   └─ 우리의 NoteCreateData 배열로 교체
```

### 4단계: 배경 영상/음악 교체

**비유**: 무대의 배경 영상과 음악을 우리가 준비한 것으로 교체

```
BGA 주입:
   VideoPlayer.time = BGM 시간 % 비디오 길이  ← 공식 5

BGM 주입:
   perfectSample = BGM 길이 × 48000  ← 공식 2
```

### 5단계: 게임 종료 시간 조정

**비유**: 원래 게임은 120초 후에 끝나지만, 우리 음악은 180초이므로 조정

```
4개 필드 조정:
   musicFadeOutEndSample = BGM 길이 × 48000  ← 공식 4
   musicFadeOutStartSample = (BGM 길이 - 1) × 48000
   screenFadeOutStartSample = (BGM 길이 - 1.5) × 48000
   screenFadeOutEndSample = BGM 길이 × 48000
```

## 📊 핵심 공식 요약표

| 공식 | 설명 | 예시 | 조건 |
|------|------|------|------|
| **Time = Tick × (240 / BPM)** | Tick을 초로 변환 (단순) | Tick=2, BPM=120 → Time=4초 | ⚠️ BPM 변화 없을 때만! |
| **Time = Σ(구간별 계산)** | Tick을 초로 변환 (복잡) | 구간별 누적 계산 | ✅ BPM 변화 있을 때 필수 |
| **perfectSample = Time × 48000** | 초를 샘플로 변환 | Time=4초 → 192,000 샘플 | 항상 적용 |
| **Duration = 끝 Time - 시작 Time** | 홀드 노트 길이 | 5초 - 2초 = 3초 | 항상 적용 |
| **BGA 시간 = BGM 시간 % 비디오 길이** | 동기화 | 65초 % 30초 = 5초 | 항상 적용 |
| **시간 차이 = \|실제 - 예상\|** | 동기화 오차 | \|5.2 - 5.0\| = 0.2초 | 항상 적용 |
| **MusicID = 54 + 앨범 인덱스** | 커스텀 ID 생성 | 앨범1 → 55 | 항상 적용 |

## 🎯 실전 예제

### 예제 1: BMS 노트 시간 계산 (BPM 변화 없음)

**문제**: BPM = 120 (변화 없음), Tick = 3.5인 노트의 perfectSample은?

**풀이**:
1. Time = 3.5 × (240 / 120) = 3.5 × 2 = **7.0초**
2. perfectSample = 7.0 × 48000 = **336,000 샘플**

---

### 예제 1-2: BMS 노트 시간 계산 (BPM 변화 있음) ⚠️ 복잡!

**문제**: 
- 기본 BPM = 120
- Tick 2.0에서 BPM → 150
- Tick 5.0에서 BPM → 100
- **Tick = 6.0인 노트의 perfectSample은?**

**풀이**:

**1단계: 구간별 시간 계산**

- **구간 1** (0 ~ 2.0마디, BPM = 120):
  - time1 = 2.0 × 4 × (60/120) = 2.0 × 2 = **4.0초**

- **구간 2** (2.0 ~ 5.0마디, BPM = 150):
  - time2 = 3.0 × 4 × (60/150) = 3.0 × 1.6 = **4.8초**

- **구간 3** (5.0 ~ 6.0마디, BPM = 100):
  - time3 = 1.0 × 4 × (60/100) = 1.0 × 2.4 = **2.4초**

**2단계: 전체 시간 합산**
- Time = 4.0 + 4.8 + 2.4 = **11.2초**

**3단계: perfectSample 계산**
- perfectSample = 11.2 × 48000 = **537,600 샘플**

**⚠️ 주의**: 단순 공식 `6.0 × (240/120) = 12.0초`를 사용하면 **틀립니다!**

### 예제 2: 홀드 노트 길이 계산

**문제**: 시작 노트 perfectSample = 96,000, 끝 노트 perfectSample = 240,000인 홀드 노트의 Duration은?

**풀이**:
1. Duration(Sample) = 240,000 - 96,000 = **144,000 샘플**
2. Duration(초) = 144,000 / 48000 = **3.0초**

### 예제 3: 게임 종료 시간 조정

**문제**: BGM 길이 = 180초일 때 4개 필드를 조정하세요.

**풀이**:
1. musicFadeOutEndSample = 180 × 48000 = **8,640,000 샘플**
2. musicFadeOutStartSample = 179 × 48000 = **8,592,000 샘플**
3. screenFadeOutStartSample = 178.5 × 48000 = **8,568,000 샘플**
4. screenFadeOutEndSample = 180 × 48000 = **8,640,000 샘플**

### 예제 4: BGA 동기화

**문제**: BGM 시간 = 95초, 비디오 길이 = 30초일 때 BGA 시간은?

**풀이**:
- BGA 시간 = 95 % 30 = **5초** (비디오의 5초 지점부터 재생)

## 💡 핵심 인사이트

1. **시간 변환의 핵심**: Tick → Time → Sample 순서로 변환
2. **단순 vs 복잡**: BPM 변화 없으면 단순 공식, 있으면 구간별 누적 계산 필수!
3. **BPM의 역할**: BPM이 빠를수록 같은 Tick을 더 빠르게 재생
4. **샘플레이트**: 게임은 48000Hz 사용 (1초 = 48000 샘플)
5. **Dictionary의 힘**: 노트가 많을수록 O(1) 검색이 필수
6. **4개 필드 필수**: 게임 종료 시간 조정은 4개 필드 모두 필요
7. **동기화의 정확도**: 0.1초 이상 차이 시 재동기화
8. **⚠️ BPM 변화 주의**: 단순 공식은 BPM이 한 번도 안 바뀔 때만 사용 가능!

이 모드는 **수학 공식을 기반으로 한 정확한 시간 계산 시스템**입니다.

- **Tick → Time → Sample**: 정확한 타이밍 계산
