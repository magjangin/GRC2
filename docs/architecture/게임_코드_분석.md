# 게임 코드 분석 문서 인덱스

이 문서는 GUNVOLT RECORDS Cychronicle 게임의 내부 코드 구조를 리버스 엔지니어링을 통해 추정한 내용을 담고 있습니다. 실제 게임 코드를 직접 확인할 수 없으므로, 모드 코드의 동작 방식과 Reflection을 통해 확인된 정보를 바탕으로 추정한 내용입니다.

- VideoPlayer 통합
- BGA/BGM 동기화 시스템
- `cMusicSelectSceneUIUpdater` 클래스 (곡 선택 화면)
- 프리뷰 BGM 주입 메커니즘
- 커스텀 아트워크 주입 시스템
- `CustomAssetManager` 클래스
- **곡 제목 기반 커스텀 차트 판단**: 원본 제목 목록과 커스텀 차트 앨범 제목을 비교하여 판단
- **일반 곡 선택 시 원본 복원**: `coOpen`에서 일반 곡 감지 시 원본 아트워크/BGM 자동 복원
- **BMS 노트 주입 가드**: 커스텀 차트가 아닐 때 BMS 노트 주입 방지 (`NoteArrayHooks.TryInjectBmsNotes()`)
- **커스텀 곡 주입 시스템**: `mCellHaviableMusicDataList`에 직접 주입
  - `initializeAllItemByCrrentMusicData` 후킹으로 커스텀 곡 추가
  - 템플릿 곡 복제하여 커스텀 곡 생성
  - 커스텀 MusicID 생성: 54-511 범위 (458개), 이후 10000 이상
  - 템플릿 곡의 제목을 원본 제목으로 등록하여 텍스트 교체 지원
- **텍스트 훅 스위치 제어**: `coOpen`에서 커스텀 차트 감지 시 스위치 활성화
  - 플레이 씬/결과 씬에서 원본 제목을 커스텀 제목으로 교체

## 주요 추정 사항 요약

1. **노트 데이터 구조**: `NoteCreateData` 클래스와 관련 Enum 타입들의 구조
2. **노트 생성 로직**: `createAllNote()` 및 `createNote()` 메서드의 동작 방식
3. **홀드/페어리 노트 처리**: `connectNodeDataArray`를 통한 연결 노트 처리
4. **BGM/BGA 관리**: `cBGMBeatManager` 및 `VideoPlayer` 통합
5. **게임 종료 로직**: 여러 조건을 고려한 종료 시간 계산

## 검증 방법

게임 코드의 실제 구조를 확인하려면:
1. **디컴파일러 사용**: ILSpy, dnSpy 등을 사용하여 `Assembly-CSharp.dll` 분석
2. **런타임 로깅**: 필요한 경우 임시 진단 코드를 별도로 추가하여 Reflection 기반 확인
3. **노트 배열 덤프 분석**: 현재 소스 기준 `NoteArrayJsonDumper`는 제거되었습니다. 덤프가 필요하면 테스트/도구 코드로 별도 구현하는 것을 권장합니다.

## 모드 시스템 기능

### Steam 게임 업데이트 차단
- **SteamManifestLocker**: Steam 매니페스트 파일을 읽기 전용으로 설정하여 게임 업데이트 차단
  - `SceneDetector.OnInitializeMelon()`에서 자동 호출
  - 게임 업데이트로 인한 모드 호환성 문제를 방지
  - Steam 매니페스트 파일 경로: `H:\steam\steamapps\appmanifest_2585040.acf` (게임별로 다를 수 있음)
  - 파일 속성을 `ReadOnly`로 설정하여 Steam이 업데이트를 시도하지 못하도록 차단
  - 로그 태그: `[GRC2] [SteamManifestLock]`, `[GRC2] [Main]`
