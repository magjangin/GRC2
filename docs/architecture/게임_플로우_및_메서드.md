# 게임 플로우 및 메서드 동작

이 문서는 GUNVOLT RECORDS Cychronicle 게임의 플로우 및 메서드 동작을 설명합니다.

## 게임 시작 → 노트 생성 플로우

```
1. 게임 시작
   ↓
2. 모드 초기화 (SceneDetector.OnInitializeMelon)
   ├─ SteamManifestLocker.LockManifest() - 게임 업데이트 차단 (자동 호출)
   └─ 기타 초기화 작업
   ↓
3. 씬 로드 (FairyModeScene 또는 PlayMovieScene)
   ↓
3. cFairyModeNotesManager 인스턴스 생성
   ↓
4. loadFairyNoteDatasJsonToArray() 호출
   ├─ JSON 파일에서 노트 데이터 로드
   └─ mFairyNoteCreateDataArray에 할당
   ↓
5. createAllNote() 호출 (NoteArrayHooks.CreateAllNotePrefix 후킹)
   ├─ TryInjectBmsNotes() 호출
   │  ├─ 커스텀 차트 선택 여부 확인 (CustomAssetManager.IsCustomChartSelected())
   │  ├─ 커스텀 차트가 아니면 주입 건너뛰고 로그 출력
   │  └─ 커스텀 차트이면 BMS 노트 주입 진행
   ├─ mFairyNoteCreateDataArray 순회
   ├─ 각 노트에 대해 createNote() 호출
   └─ 노트 오브젝트 생성 및 배치
   ↓
6. 게임 플레이 시작
   ├─ 노트가 perfectSample 시간에 생성됨
   └─ 플레이어 입력 처리
```

## 노트 생성 상세 플로우

**`createNote(NoteCreateData createData)` 내부 동작 추정:**

```csharp
public void createNote(NoteCreateData createData)
{
    // 1. 노트 타입 확인
    NoteTypeId noteType = createData.noteTypeID;
    
    // 2. 노트 오브젝트 생성
    GameObject noteObject = null;
    switch (noteType)
    {
        case NoteTypeId.Touch:
            noteObject = CreateTouchNote(createData);
            break;
        case NoteTypeId.Hold:
            noteObject = CreateHoldNote(createData);
            break;
        case NoteTypeId.Flick:
            noteObject = CreateFlickNote(createData);
            break;
        case NoteTypeId.Fairy:
            noteObject = CreateFairyNote(createData);
            break;
    }
    
    // 3. connectNodeDataArray 처리 (홀드/페어리 노트)
    if (createData.connectNodeDataArray != null && createData.connectNodeDataArray.Length > 0)
    {
        // 연결 노트 생성 및 연결
        foreach (var connectNode in createData.connectNodeDataArray)
        {
            GameObject connectNoteObject = CreateConnectNote(connectNode);
            LinkNoteToConnectNode(noteObject, connectNoteObject);
        }
    }
    
    // 4. 노트를 게임 월드에 배치
    PlaceNoteInWorld(noteObject, createData.perfectSample);
    
    // 5. 상태 플래그 설정
    createData.isCreated = true;
}
```

## NoteCreateData 생성자 파라미터

모드는 `NoteConstructorHelper.TryFindConstructor()`로 여러 생성자 시그니처(2~6개 파라미터)를 순서대로 시도하고, 실패 시 기본 생성자로 인스턴스를 만든 뒤 `FieldAccessHelper`·`NoteFieldInitializer`로 필드를 설정합니다.

**생성자 로깅:**
- 생성자 호출 성공 시 로그는 `NoteConstructorHelper.EnableConstructorLogging`(기본값 false)로 제어됩니다.
- 게임의 `NoteCreateData` 생성자 목록과 길이/방향 관련 필드를 보려면 `NoteConstructorHelper.LogNoteCreateDataConstructorsAndFields()`를 호출하면 됩니다.

**참고:**
- 생성자 탐색 순서: (time,lane), (sample,lane) → (time,lane,type) → (time,lane,type,leftRight) → (lane,time,sample,type,leftRight) → 6개 파라미터(방향 포함/기본값).
- 인스턴스 생성 후 `NoteFieldInitializer`로 directionIndex, turnDireciton, noteSize, slideEndFlickDirection, Boolean 필드 등을 설정합니다.

## 홀드 노트 처리 로직 추정

```csharp
private GameObject CreateHoldNote(NoteCreateData createData)
{
    // 1. 홀드 시작 노트 생성
    GameObject holdStart = CreateHoldStartNote(createData);
    
    // 2. connectNodeDataArray 확인
    if (createData.connectNodeDataArray != null && createData.connectNodeDataArray.Length > 0)
    {
        // 3. 끝 노트 데이터 추출
        NoteCreateData endNoteData = createData.connectNodeDataArray[0];
        
        // 4. 홀드 끝 노트 생성
        GameObject holdEnd = CreateHoldEndNote(endNoteData);
        
        // 5. 시작 노트와 끝 노트 연결
        LinkHoldStartToEnd(holdStart, holdEnd);
        
        // 6. 홀드 길이 계산
        float holdDuration = (endNoteData.perfectSample - createData.perfectSample) / 48000f;
        SetHoldDuration(holdStart, holdDuration);
    }
    
    return holdStart;
}
```

## 페어리 노트 처리 로직 추정

```csharp
private GameObject CreateFairyNote(NoteCreateData createData)
{
    // 1. 페어리 시작 노트 생성
    GameObject fairyStart = CreateFairyStartNote(createData);
    
    // 2. turnDireciton 설정
    string turnDirection = createData.turnDireciton;  // "Left" 또는 "Right"
    SetFairyTurnDirection(fairyStart, turnDirection);
    
    // 3. connectNodeDataArray 확인
    if (createData.connectNodeDataArray != null && createData.connectNodeDataArray.Length > 0)
    {
        // 4. 끝 노트 데이터 추출
        NoteCreateData endNoteData = createData.connectNodeDataArray[0];
        
        // 5. 페어리 끝 노트 생성 (turnDireciton 복사)
        endNoteData.turnDireciton = createData.turnDireciton;  // 시작 노트와 동일
        GameObject fairyEnd = CreateFairyEndNote(endNoteData);
        
        // 6. 시작 노트와 끝 노트 연결
        LinkFairyStartToEnd(fairyStart, fairyEnd);
        
        // 7. 페어리 길이 계산
        float fairyDuration = (endNoteData.perfectSample - createData.perfectSample) / 48000f;
        SetFairyDuration(fairyStart, fairyDuration);
    }
    
    return fairyStart;
}
```

## 곡 선택 화면 시스템

### 곡 선택 UI 클래스

- **`cMusicSelectSceneUIUpdater`**: 곡 선택 화면 UI 업데이트 관리
  - `noticeChangedMusic()`: 곡 선택 변경 시 호출되는 메서드
  - `openSortWindow()`, `openFilterWindow()`: 정렬/필터 창 열기 메서드
- **`cMusicSelectSceneSelectingMusicUI`**: 곡 선택 UI 관리
  - `coOpen()`: 곡 선택 창 열기 코루틴
    - `mCurrentDispData.mMusicSelectData.musicID`를 통해 현재 선택된 곡의 MusicID 확인
    - 커스텀 차트 감지 및 아트워크/BGM 로드에 사용
  - `coClose()`: 곡 선택 창 닫기 코루틴

### 곡 선택 플로우

```
1. 곡 선택 화면 진입
   ↓
2. 곡 목록 초기화 (initializeAllItemByCrrentMusicData 후킹)
   ├─ mCellHaviableMusicDataList에 커스텀 곡 주입
   ├─ 템플릿 곡 복제하여 커스텀 곡 생성
   ├─ 각 앨범마다 고유한 커스텀 MusicID 생성 (54-511, 10000 이상)
   └─ 템플릿 곡의 제목을 원본 제목으로 등록
   ↓
3. 곡 목록 표시 (커스텀 곡 포함)
   ↓
4. 곡 선택 시:
   ├─ noticeChangedMusic() 호출 (AudioClipPatch.NoticeChangedMusicPostfix 후킹)
   └─ coOpen() 호출 (GameFlowHooks.CoOpenPrefix (구현: GameFlowHooks.Navigation.cs) 후킹)
   ↓
5. 곡 시작 전 윈도우 열기 (coOpenPreMusicStartWindow 후킹):
   ├─ 현재 선택된 앨범의 캐릭터(아티스트 ID) 확인
   │  ├─ SongInfo.Character 우선 확인
   │  └─ 없으면 SongInfo.Artist 사용
   ├─ 해당 아티스트의 첫 곡 정보 조회 (AlbumManager.GetArtistFirstSong (구현: AlbumManager.Mappings.cs))
   ├─ MusicID를 첫 곡 MusicID로 변경 (mCurentMusicId, mCurrentMusicID 필드 업데이트)
   ├─ 원본 제목 등록 (AlbumManager.RegisterOriginalTitle (구현: AlbumManager.Mappings.cs))
   ├─ 아티스트 ID 저장 (AlbumManager.SetCurrentArtistId (구현: AlbumManager.Mappings.cs))
   └─ 아티스트 ID로 변경한 경우 커스텀 차트 처리(FIRST로 변경) 건너뛰기
   ↓
6. 커스텀 차트 감지 (coOpen 후킹):
   ├─ mCurrentDispData.mMusicSelectData.musicID 및 songTitle 확인
   ├─ 곡 제목 기반 판단:
   │  ├─ 곡 제목이 원본 제목 목록에 있으면 → 일반 곡
   │  ├─ 곡 제목이 커스텀 차트 앨범 제목이면 → 커스텀 차트
   │  └─ 둘 다 아니면 MusicID로 확인
   ├─ 커스텀 차트인 경우:
   │  ├─ AlbumManager.SelectAlbumByMusicID() (구현: AlbumManager.Mappings.cs) - 앨범 선택
   │  ├─ CustomBgmPlayer.InjectCustomBgm() - 커스텀 프리뷰 BGM 주입
   │  ├─ PreviewAudioManager.StopPreviewAndAmbient() - 원본 프리뷰 중지
   │  ├─ CustomAssetManager.LoadCustomArtwork() - 커스텀 아트워크 로드
   │  └─ TextPatch.EnableTextReplacement(true) - 텍스트 훅 스위치 ON
   ├─ 일반 곡인 경우:
   │  ├─ CustomBgmPlayer.CleanupAndRestore() - 원본 프리뷰 복원
   │  └─ TextPatch.EnableTextReplacement(false) - 텍스트 훅 스위치 OFF
   │  └─ TextPatch.EnableTextReplacement(true) - 텍스트 훅 스위치 ON
   ├─ 일반 곡인 경우:
   │  ├─ CustomBgmPlayer.CleanupAndRestore() - 원본 프리뷰 복원
   │  └─ TextPatch.EnableTextReplacement(false) - 텍스트 훅 스위치 OFF
   │  └─ ArtworkUpdater.UpdateArtwork() - 커스텀 아트워크 업데이트
   └─ 일반 곡인 경우:
      ├─ CustomAssetManager.SetCustomChartSelected(false) - 커스텀 차트 선택 해제
      ├─ CustomBgmPlayer.CleanupAndRestore() - 원본 아트워크/BGM 복원
      └─ **중요**: BMS 노트 주입도 이 플래그를 확인하여 커스텀 차트가 아닐 때 주입하지 않음
```

## 씬 관리 시스템

### 씬 구조

게임은 Unity 씬 시스템을 사용하며, 다음과 같은 씬들이 존재합니다:

- **FairyModeScene**: 페어리 모드 플레이 씬
- **PlayMovieScene**: 영상 재생 씬 (추정)
- **MusicSelectScene**: 곡 선택 화면
- 기타 메뉴/결과 화면 씬들

### 씬 로드 시 동작 추정

```csharp
// 씬 로드 시 호출되는 초기화 로직 (추정)
public class SceneInitializer : MonoBehaviour
{
    void Start()
    {
        // 1. cFairyModeNotesManager 인스턴스 찾기
        var notesManager = FindObjectOfType<cFairyModeNotesManager>();
        
        // 2. 노트 데이터 로드
        notesManager.loadFairyNoteDatasJsonToArray();
        
        // 3. 노트 생성
        notesManager.createAllNote();
        
        // 4. BGM/BGA 초기화
        InitializeBGM();
        InitializeBGA();
    }
}
```

## 성능 최적화 관련 추정

### 게임의 노트 처리 최적화

게임은 다음과 같은 최적화를 사용할 것으로 추정됩니다:

1. **노트 풀링**: 노트 오브젝트를 재사용하여 메모리 할당 최소화
2. **시간 기반 정렬**: `perfectSample` 기준으로 정렬된 배열 사용
3. **비활성 노트 제거**: 화면 밖으로 나간 노트는 자동으로 제거

### 모드의 최적화 전략

모드는 게임과의 상호작용을 최소화하기 위해 다음 최적화를 사용합니다:

1. **필드 캐싱**: `FieldAccessHelper`를 통한 Reflection 캐싱
2. **Dictionary 기반 검색**: O(1) 검색을 위한 Dictionary 사용
3. **캐시 무효화**: 참조 비교를 통한 정확한 캐시 관리

## 오토 플레이 시스템

### 오토 플레이 관련 클래스

- **`cFairyModeNotesManager`**: 노트 관리 및 오토 플레이 제어
  - `setIsAutoPlay(bool isAuto)`: 오토 플레이 설정 메서드
    - `mIsCurrentAutoPlay` 필드에 오토 플레이 상태 저장
    - `isAuto`가 `true`인 경우 `finalizeInput()` 호출
    - `isAuto`가 `false`인 경우 `initializeInput()` 호출
  - `mIsCurrentAutoPlay`: 현재 오토 플레이 상태를 저장하는 필드

### 오토 플레이 후킹

- **`GameFlowHooks.SetIsAutoPlayPrefix/Postfix()`**: `setIsAutoPlay()` 메서드 후킹
  - 오토 플레이 설정 시 로그 출력
  - 현재는 모니터링만 수행하며, 강제 활성화 기능은 비활성화됨
