# 게임 아키텍처 개요

이 문서는 GUNVOLT RECORDS Cychronicle 게임의 내부 코드 구조를 리버스 엔지니어링을 통해 추정한 내용을 담고 있습니다. 실제 게임 코드를 직접 확인할 수 없으므로, 모드 코드의 동작 방식과 Reflection을 통해 확인된 정보를 바탕으로 추정한 내용입니다.

## 네임스페이스 구조

게임은 `IntiCreates` 네임스페이스 하위에 다음과 같은 구조로 구성되어 있습니다:

```
IntiCreates
├── RythmGame
│   ├── FairyMode
│   │   ├── NoteCreateData
│   │   ├── NoteTypeId (Enum)
│   │   ├── NoteLaneLeftRight (Enum)
│   │   └── NoteSubLaneType (Enum)
│   ├── NoteDirectionIndex (Enum)
│   ├── NoteSize (Enum)
│   └── SlideEndFlickDirection (Enum)
└── cFairyModeNotesManager
└── cBGMBeatManager
```

## Assembly 구조

- **Assembly-CSharp.dll**: 게임의 핵심 로직이 포함된 메인 어셈블리
- 위치: `GUNVOLT_RECORDS_Cychronicle_Data/Managed/Assembly-CSharp.dll`
- Unity MonoBehaviour 기반의 게임 로직 포함

## 모드 아키텍처

모드는 다음과 같은 구조로 구성되어 있습니다:

### 핵심 네임스페이스
- **GRC2.Core**: 핵심 기능 (SceneDetector, AlbumManager, MusicInjector, ReflectionHelper, ResultSceneInjector 등)
- **GRC2.Injectors**: BGM/BGA 주입 및 동기화 (BgmBgaInjector, BgaBgmSyncManager, BgmAudioStateChecker 등)
- **GRC2.Harmony.Handlers**: Harmony Prefix/Postfix 본문 및 연동 유틸 (곡 목록, 프리뷰 BGM, 아트워크 등)
- **GRC2.Harmony.Registration**: Harmony에 패치를 등록하는 클래스 (`*Patcher`)
- **GRC2.Parsers**: BMS 파일 파싱
- **GRC2.Converters**: BMS 노트를 게임 노트로 변환
- **GRC2.Builders**: NoteCreateData 생성
- **GRC2.Processors**: 홀드/페어리 노트 처리
- **GRC2.Harmony.Hooks**: 게임 메서드에 붙는 훅 진입점 (NoteArrayHooks, GameFlowHooks, MusicScrollViewHooks)
- **GRC2.Services**: 커스텀 차트 감지·에셋 로드 서비스 (CustomChartHandler)
- **GRC2.Helpers**: 유틸리티 클래스 (ErrorLogger, FieldAccessHelper 등)

### 주요 컴포넌트
- **CustomAssetManager**: 커스텀 아트워크와 BGM 관리
- **PlaySceneArtworkInjector**: 플레이 씬 아트워크 주입
- **ResultSceneInjector**: 결과 씬(RythmGameResultScene) 아트워크·난이도 주입
- **SceneHandler**: 씬별 처리 로직
- **ReflectionHelper**: 리플렉션 헬퍼
- **CustomChartHandler** (Services): coOpen 등에서 호출되는 커스텀 차트 감지 및 아트워크/BGM 로드
