# 게임 클래스 구조

이 문서는 GUNVOLT RECORDS Cychronicle 게임의 핵심 클래스 및 데이터 구조를 설명합니다.

## 핵심 클래스 구조

### 1. `cFairyModeNotesManager`

페어리 모드의 노트 관리를 담당하는 핵심 클래스입니다.

#### 추정 클래스 구조

```csharp
namespace IntiCreates
{
    public class cFairyModeNotesManager : MonoBehaviour
    {
        // 노트 배열 필드
        private NoteCreateData[] mFairyNoteCreateDataArray;  // 페어리 모드 노트 배열
        
        // 게임 종료 관련 필드 (추정)
        private float mMaxTiming;           // 최대 타이밍 (마지막 노트 시간)
        private float mBgmLength;          // BGM 길이 (초)
        private float mCruiseFinishTime;   // 크루즈 모드 종료 시간 (초)
        
        // 메서드
        public void createAllNote()        // 모든 노트 생성
        public void createNote(NoteCreateData createData)  // 단일 노트 생성
        public void loadFairyNoteDatasJsonToArray()  // JSON에서 노트 데이터 로드
    }
}
```

#### 주요 메서드 동작 추정

**`createAllNote()` 메서드:**
```csharp
public void createAllNote()
{
    // 1. mFairyNoteCreateDataArray를 순회
    foreach (var noteData in mFairyNoteCreateDataArray)
    {
        // 2. 각 노트에 대해 createNote 호출
        createNote(noteData);
    }
    
    // 3. 게임 종료 시간 계산 (추정)
    CalculateFinishTime();
}
```

**`createNote(NoteCreateData createData)` 메서드:**
```csharp
public void createNote(NoteCreateData createData)
{
    // 1. NoteCreateData에서 노트 정보 추출
    int perfectSample = createData.perfectSample;
    NoteTypeId noteType = createData.noteTypeID;
    NoteDirectionIndex direction = createData.directionIndex;
    
    // 2. 노트 타입에 따라 적절한 노트 오브젝트 생성
    GameObject noteObject = CreateNoteObject(noteType, direction);
    
    // 3. connectNodeDataArray가 있으면 연결 노트도 생성
    if (createData.connectNodeDataArray != null)
    {
        foreach (var connectNode in createData.connectNodeDataArray)
        {
            CreateConnectNode(noteObject, connectNode);
        }
    }
    
    // 4. 노트를 게임 월드에 배치
    PlaceNoteInWorld(noteObject, perfectSample);
}
```

**`loadFairyNoteDatasJsonToArray()` 메서드:**
```csharp
public void loadFairyNoteDatasJsonToArray()
{
    // 1. JSON 파일에서 노트 데이터 로드
    string jsonPath = GetJsonPath();
    string jsonContent = File.ReadAllText(jsonPath);
    
    // 2. JSON을 NoteCreateData 배열로 역직렬화
    NoteCreateData[] notes = JsonUtility.FromJson<NoteCreateData[]>(jsonContent);
    
    // 3. mFairyNoteCreateDataArray에 할당
    mFairyNoteCreateDataArray = notes;
}
```

## 노트 데이터 구조

### `NoteCreateData` 클래스

게임의 노트 데이터를 담는 핵심 데이터 클래스입니다.

#### 추정 클래스 구조

```csharp
namespace IntiCreates.RythmGame.FairyMode
{
    public class NoteCreateData
    {
        // 시간 관련
        public int perfectSample;  // 노트 생성 시간 (샘플 단위, 48000Hz 기준)
        
        // 레인 관련
        public NoteLaneLeftRight laneLeftRightID;  // 왼쪽/오른쪽 레인
        public NoteSubLaneType subLaneID;          // 서브 레인 (Lane_1, Lane_2, Lane_3)
        
        // 노트 타입 및 방향
        public NoteTypeId noteTypeID;              // 노트 타입 (Touch, Hold, Flick, Fairy)
        public NoteDirectionIndex directionIndex;   // 방향 인덱스 (8방향)
        public NoteSize noteSize;                   // 노트 크기 (Scale1 등)
        
        // 페어리 노트 전용
        public string turnDireciton;               // 회전 방향 ("Left" 또는 "Right")
        
        // 플릭 관련
        public SlideEndFlickDirection slideEndFlickDirection;  // 슬라이드 끝 플릭 방향
        
        // 연결 노트 배열 (홀드/페어리 노트의 끝 노트)
        public NoteCreateData[] connectNodeDataArray;  // 연결된 노트 데이터 배열
        
        // 상태 플래그
        public bool isCritical;                    // 크리티컬 노트 여부
        public bool isCreated;                     // 생성 완료 여부
        public bool isLast;                        // 마지막 노트 여부
        public bool isThisBoostStart;              // 부스트 시작 여부
        public bool isSameTimingWithFlick;         // 플릭과 같은 타이밍 여부
        public bool isNearByFrontFlick4TapOrHold;  // 앞쪽 플릭과 근접 여부
        public bool isNearByFrontSlideEnd4Flick;   // 앞쪽 슬라이드 끝 플릭과 근접 여부
    }
}
```

#### 필드 상세 설명

**`perfectSample` (int):**
- 노트가 생성되어야 하는 정확한 시간을 샘플 단위로 표현
- 샘플레이트: 48000Hz (1초 = 48000 샘플)
- 변환 공식: `perfectSample = (int)(timeInSeconds * 48000)`

**`connectNodeDataArray` (NoteCreateData[])`:**
- 홀드/페어리 노트의 끝 노트를 담는 배열
- 홀드 노트: 시작 노트(02)의 `connectNodeDataArray`에 끝 노트(19)가 포함됨
- 페어리 노트: 시작 노트(11-18)의 `connectNodeDataArray`에 끝 노트(1A-1B)가 포함됨
- 게임은 이 배열을 확인하여 홀드/페어리 노트를 인식

**`turnDireciton` (string):**
- 페어리 노트 전용 필드
- 값: `"Left"` 또는 `"Right"`
- 페어리 노트의 회전 방향을 결정
- `directionIndex`의 문자열 표현에서 "LEFT" 또는 "RIGHT"를 추출하여 자동 설정

## Enum 타입 구조

### 1. `NoteTypeId` Enum

```csharp
namespace IntiCreates.RythmGame.FairyMode
{
    public enum NoteTypeId
    {
        Touch,  // 일반 노트 (BMS: 01)
        Hold,   // 홀드 노트 (BMS: 02, 19)
        Flick,  // 플릭 노트 (BMS: 03-0A)
        Fairy   // 페어리 노트 (BMS: 11-18, 1A-1B)
    }
}
```

### 2. `NoteLaneLeftRight` Enum

```csharp
namespace IntiCreates.RythmGame.FairyMode

    }
```

### 3. `NoteSubLaneType` Enum

```csharp
namespace IntiCreates.RythmGame.FairyMode
{
    public enum NoteSubLaneType
    {
        Lane_1,  // 첫 번째 레인 (BMS 채널 16/14)
        Lane_2,  // 두 번째 레인 (BMS 채널 11/15)
        Lane_3   // 세 번째 레인 (BMS 채널 12/18)

```

### 4. `NoteDirectionIndex` Enum

```csharp
namespace IntiCreates.RythmGame
{
    public enum NoteDirectionIndex
    {
        CENTER_MIDDLE,  // 중앙 중간 (기본값)
        CENTER_TOP,     // 중앙 위
        CENTER_BOTTOM,  // 중앙 아래
        LEFT_BOTTOM,    // 왼쪽 아래
        LEFT_MIDDLE,    // 왼쪽 중간
        LEFT_TOP,       // 왼쪽 위
        RIGHT_BOTTOM,   // 오른쪽 아래
        RIGHT_MIDDLE,   // 오른쪽 중간
        RIGHT_TOP       // 오른쪽 위
    }
}
```

**BMS → 게임 방향 매핑:**
- `NoteDirection.Left` → `LEFT_MIDDLE` (왼쪽 중간)
- `NoteDirection.LeftUp` → `LEFT_TOP` (왼쪽 위)
- `NoteDirection.Up` → `CENTER_TOP` (중앙 위)
- `NoteDirection.RightUp` → `RIGHT_TOP` (오른쪽 위)
- `NoteDirection.Right` → `RIGHT_MIDDLE` (오른쪽 중간)
- `NoteDirection.RightDown` → `RIGHT_BOTTOM` (오른쪽 아래)
- `NoteDirection.Down` → `CENTER_BOTTOM` (중앙 아래)
- `NoteDirection.LeftDown` → `LEFT_BOTTOM` (왼쪽 아래)

**레인 기반 자동 방향 설정 (홀드 노트):**
- 왼쪽 레인:
  - Lane 0 → `LEFT_BOTTOM`
  - Lane 1 → `LEFT_MIDDLE`
  - Lane 2 → `LEFT_TOP`
- 오른쪽 레인:
  - Lane 0 → `RIGHT_BOTTOM`
  - Lane 1 → `RIGHT_MIDDLE`
  - Lane 2 → `RIGHT_TOP`
