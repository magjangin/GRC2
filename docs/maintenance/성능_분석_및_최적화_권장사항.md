## 🔴 심각한 성능 문제 (우선순위 높음)

### 1. **ReflectionHelper의 비효율적 어셈블리 순회**
**위치**: [GRC2\Core\Infrastructure\ReflectionHelper.cs](GRC2/Core/Infrastructure/ReflectionHelper.cs)

**문제점**:
```csharp
public static Type FindType(string typeName)
{
    foreach (var assembly in AppDomain.CurrentDomain.GetAssemblies())
    {
        try
        {
            var type = assembly.GetType(typeName);
            if (type != null)
            {
                return type;
            }
        }
        catch { }
    }
    return null;
}
```

- **매번 모든 로드된 어셈블리를 순회** (보통 30~50개)
- 타입 찾기 실패 시 모든 어셈블리를 검사 후 반환
- 게임 루프 중 자주 호출될 경우 누적 성능 저하

**개선안**:
- 이미 발견된 타입을 캐시 (Dictionary<string, Type>)
- AppDomain.CurrentDomain.GetAssemblies() 결과 캐싱
- 두 번째 호출부터 O(1) 조회 가능

**예상 개선**: 타입 검색 시간 **10배 이상 단축**

### 2. **HoldNoteProcessor의 중첩 반복 및 Reflection 오버헤드**
**위치**: [GRC2\Processors\HoldNoteProcessor.cs](GRC2/Processors/HoldNoteProcessor.cs) (Line 78~150)

**문제점**:
```csharp
foreach (var holdStartBms in holdStartBmsNotes)  // 홀드 노트 수만큼 반복
{
    foreach (var noteObj in noteList)  // 전체 노트 수만큼 반복
    {
        // 매 반복마다 Reflection으로 필드 가져오기
        var perfectSampleField = Loaders.GameTypeLoader.NoteCreateDataType.GetField(...);
        var laneLeftRightField = Loaders.GameTypeLoader.NoteCreateDataType.GetField(...);
        var subLaneField = Loaders.GameTypeLoader.NoteCreateDataType.GetField(...);
        var noteTypeIdField = Loaders.GameTypeLoader.NoteCreateDataType.GetField(...);
    }
}
```

**성능 분석**:
- 빅오 복잡도: **O(n × m × 4)** (n: 홀드 노트, m: 전체 노트, 4: 필드 검색)
- 예: 홀드 노트 500개 × 노트 2000개 × 4 필드 = **400만 번의 GetField() 호출**
- GetField()는 문자열 기반 리플렉션으로 매우 느림

**현재 캐싱**:
- FieldAccessHelper에서 필드 캐시 제공하지만, **내부 루프에서 효과 제한적**

**개선안**:
```csharp
// 루프 외부에서 한 번만 필드 정보 가져오기
var perfectSampleField = FieldAccessHelper.GetCachedField("perfectSample");
var laneLeftRightField = FieldAccessHelper.GetCachedField("laneLeftRightID");
var subLaneField = FieldAccessHelper.GetCachedField("subLaneID");
var noteTypeIdField = FieldAccessHelper.GetCachedField("noteTypeID");

foreach (var holdStartBms in holdStartBmsNotes)
{
    foreach (var noteObj in noteList)
    {
        // 이미 캐시된 필드 정보 직접 사용
        var perfectSample = perfectSampleField?.GetValue(noteObj);
        // ...
    }
}
```

**예상 개선**: 최악의 경우 **50~100배 성능 향상**

### 4. **BgmInjector의 과도한 FindObjectsOfType 호출**
**위치**: [GRC2\Injectors\Bgm\BgmInjector.cs](GRC2/Injectors/Bgm/BgmInjector.cs) (Line 97~150)

**문제점**:
```csharp
private static IEnumerator FindBgmBeatManagerInstance(...)

    // 매 코루틴 반복마다 전체 씬의 모든 객체 검색
    var bgmManagers = UnityEngine.Object.FindObjectsOfType(bgmBeatManagerType);

    // AudioSource도 전체 검색
```

- 각 호출마다 메모리 할당 발생

**개선안**:
```csharp
// 첫 번째 발견 후 캐시

{
    _bgmBeatManagerInstance = UnityEngine.Object.FindObjectOfType(bgmBeatManagerType);
}

// 캐시된 인스턴스 재사용
if (_bgmBeatManagerInstance != null)
{
    // 사용
}
```

**예상 개선**: 최대 **10배 감소** (중복 검색 제거)

### 5. **AlbumManager.ScanAlbumFolder의 비효율적 파일 검색**
**위치**: [GRC2\Core\Album\AlbumManager.cs](GRC2/Core/Album/AlbumManager.cs) (Line 113~145)

**문제점**:
```csharp
private static AlbumInfo ScanAlbumFolder(string albumFolderPath)
{
    // 각 파일 확장자별로 별도의 GetFiles 호출
    albumInfo.BmsFiles = Directory.GetFiles(..., "*.bms", SearchOption.TopDirectoryOnly)
        .Concat(Directory.GetFiles(..., "*.bme", SearchOption.TopDirectoryOnly))
        .Concat(Directory.GetFiles(..., "*.bml", SearchOption.TopDirectoryOnly))
        .ToList();
    
    albumInfo.ImageFiles = Directory.GetFiles(..., "*.jpg", SearchOption.TopDirectoryOnly)
        .Concat(Directory.GetFiles(..., "*.png", SearchOption.TopDirectoryOnly))
        .Concat(Directory.GetFiles(..., "*.jpeg", SearchOption.TopDirectoryOnly))
        .ToList();
    
    // 같은 방식으로 반복...
    albumInfo.BgaFiles = Directory.GetFiles(..., "*.mp4", SearchOption.TopDirectoryOnly).ToList();
    
    albumInfo.BgmFiles = Directory.EnumerateFiles(..., "*.*", SearchOption.TopDirectoryOnly)
        .Where(f => { ... })
        .ToList();
}
```

**문제 분석**:
- 단일 폴더 검색을 **7번 분리** 수행
- 각 GetFiles는 파일시스템 접근 (I/O 오버헤드)
- 앨범 수 × 7 = 많은 I/O 작업

**개선안**:
```csharp
// 한 번의 EnumerateFiles로 모든 파일 수집
var allFiles = Directory.EnumerateFiles(albumFolderPath, "*.*", SearchOption.TopDirectoryOnly);
var bmsExt = new HashSet<string> { ".bms", ".bme", ".bml" };
var imgExt = new HashSet<string> { ".jpg", ".png", ".jpeg" };
var audioExt = new HashSet<string> { ".mp3", ".wav", ".ogg" };

foreach (var file in allFiles)
{
    var ext = Path.GetExtension(file).ToLowerInvariant();
    if (bmsExt.Contains(ext)) albumInfo.BmsFiles.Add(file);
    else if (imgExt.Contains(ext)) albumInfo.ImageFiles.Add(file);
    else if (ext == ".mp4") albumInfo.BgaFiles.Add(file);
    else if (audioExt.Contains(ext)) albumInfo.BgmFiles.Add(file);
    else if (ext == ".txt") albumInfo.TxtFile = file;
}
```

**예상 개선**: 파일 시스템 호출 **70~80% 감소**
