# 게임 종료 로직 및 모드 상호작용

이 문서는 GUNVOLT RECORDS Cychronicle 게임의 종료 로직과 모드의 상호작용을 설명합니다.

## 게임 종료 조건

게임은 다음 조건 중 하나가 만족되면 종료됩니다:

1. **마지막 노트 시간 + 여유 시간**: `mMaxTiming + 5.0초`
2. **BGM 길이**: `mBgmLength`
3. **크루즈 모드 종료 시간**: `mCruiseFinishTime`
4. **mRythmGameMusicData 필드들** (실제 게임 종료에 사용되는 필드):
   - `musicFadeOutEndSample`: BGM 페이드 아웃 종료 시간 (샘플)
   - `musicFadeOutStartSample`: BGM 페이드 아웃 시작 시간 (샘플)
   - `screenFadeOutStartSample`: 화면 페이드 아웃 시작 시간 (샘플)
   - `screenFadeOutEndSample`: 화면 페이드 아웃 종료 시간 (샘플)

### 추정 코드

```csharp
public class GameFinishChecker : MonoBehaviour
{
    private cFairyModeNotesManager notesManager;
    private cBGMBeatManager bgmManager;
    
    void Update()
    {
        float currentTime = GetCurrentPlayTime();
        
        // 종료 시간 계산
        float finishTime = Mathf.Max(
            notesManager.mMaxTiming + 5.0f,      // 마지막 노트 + 여유 시간
            bgmManager.bgmLength,                // BGM 길이
            notesManager.mCruiseFinishTime        // 크루즈 모드 종료 시간
        );
        
        // 종료 조건 확인
        if (currentTime >= finishTime)
        {
            GoToResultScreen();
        }
    }
}
```

## 게임 종료 시간 자동 계산 추정

게임이 `mFairyNoteCreateDataArray`를 분석하여 자동으로 `mMaxTiming`을 계산할 가능성이 있습니다:

```csharp
private void CalculateFinishTime()
{
    if (mFairyNoteCreateDataArray == null || mFairyNoteCreateDataArray.Length == 0)
        return;
    
    float maxTime = 0f;
    
    foreach (var note in mFairyNoteCreateDataArray)
    {
        // 일반 노트의 시간
        float noteTime = note.perfectSample / 48000f;
        maxTime = Mathf.Max(maxTime, noteTime);
        
        // connectNodeDataArray의 끝 노트 시간도 고려
        if (note.connectNodeDataArray != null)
        {
            foreach (var connectNode in note.connectNodeDataArray)
            {
                float connectTime = connectNode.perfectSample / 48000f;
                maxTime = Mathf.Max(maxTime, connectTime);
            }
        }
    }
    
    mMaxTiming = maxTime;
}
```

## 마지막 노트 플래그 설정

게임은 `isLast` 플래그를 사용하여 마지막 노트를 식별할 수 있습니다. 모드는 변환 과정에서 자동으로 마지막 노트를 찾아 `isLast`를 `true`로 설정합니다.

**모드의 마지막 노트 설정 로직:**

```csharp
// BmsNoteConverter.SetLastNoteFlag() (구현: BmsNoteConverter.Validation.cs) 메서드
private static void SetLastNoteFlag(List<object> noteList)
{
    int maxPerfectSample = int.MinValue;
    object lastNote = null;
    
    // 모든 노트와 connectNodeDataArray의 끝 노트를 확인
    foreach (var noteObj in noteList)
    {
        // 메인 노트의 perfectSample 확인
        int sample = GetPerfectSample(noteObj);
        if (sample > maxPerfectSample)
        {
            maxPerfectSample = sample;
            lastNote = noteObj;
        }
        
        // connectNodeDataArray의 끝 노트도 확인
        // (홀드/페어리 노트의 끝 노트가 마지막일 수 있음)
        foreach (var connectNode in GetConnectNodes(noteObj))
        {
            int connectSample = GetPerfectSample(connectNode);
            if (connectSample > maxPerfectSample)
            {
                maxPerfectSample = connectSample;
                lastNote = connectNode;
            }
        }
    }
    
    // 마지막 노트에 isLast 설정
    if (lastNote != null)
    {
        SetFieldValue(lastNote, "isLast", true);
    }
}
```

**핵심 포인트:**
- ✅ 모든 노트의 `perfectSample`을 비교하여 가장 큰 값을 가진 노트를 찾음
- ✅ `connectNodeDataArray`의 끝 노트도 고려 (홀드/페어리 노트의 끝 노트가 마지막일 수 있음)
- ✅ 마지막 노트에 `isLast = true` 설정
- ✅ 게임이 `isLast` 플래그를 확인하여 마지막 노트를 인식할 수 있음

## 모드와 게임의 상호작용

### Harmony 패치 포인트

모드는 Harmony를 사용하여 다음 메서드들을 후킹합니다:

#### 노트 배열 관련 후킹 (`NoteArrayHooks`)

1. **`createAllNote()` Prefix**
   - Prefix: BMS 노트 주입 (`NoteArrayHooks.CreateAllNotePrefix`)
     - `InjectBmsNotes(__instance)`: BMS 노트를 NoteCreateData로 변환하여 `mFairyNoteCreateDataArray`에 주입
2. **`loadFairyNoteDatasJsonToArray()` Prefix**
   - Prefix: BMS 노트 주입 (`NoteArrayHooks.LoadFairyNoteDatasJsonToArrayPrefix`)
     - `InjectBmsNotes(__instance)`: BMS 노트를 NoteCreateData로 변환하여 mFairyNoteCreateDataArray에 주입

#### BGM 관련 후킹 (`BgmInjectorHooks`, `BgmMethodCallHooks`)

3. **`cBGMBeatManager` 메서드 후킹** (`BgmInjectorHooks.Initialize()`)
   - `setClip`: postfix 후킹 (`BgmMethodCallHooks.MethodCallPostfix/PostfixVoid`)
     - `HandleImportantMethodCall()`: 주입된 BGM 감지 및 검증
   - `requestLoadBGM`: postfix 후킹
   - `requestPlayAudio`: postfix 후킹 (`BgmMethodCallHooks.HandleImportantMethodCall()`)
     - 주입된 BGM 감지 (클립 이름 확인)
     - `BgmFinishTimeManager.SetFinishTime()` 재호출 (BGM 길이 재확인)
     - `BgmMonitorCoroutine.StartBgmMonitorCoroutine()`로 BGM 모니터링 시작
   - `requestPause`: postfix 후킹 (로그 제외)
   - `setBPM`, `setSample`, `getAudioClip`, `isReadyPlay`: postfix 후킹
   - `setTime`: prefix 후킹 (`BgmMethodCallHooks.SetTimePrefix`, 시간 제한 적용)
     - `BgmFinishTimeManager.GetTargetFinishTime()`로 목표 시간 확인
     - 목표 시간을 초과하는 경우 목표 시간으로 제한

4. **`cRythmGameManager` 게임 종료 관련 메서드 후킹** (`BgmInjectorHooks.Initialize()`)
   - `requestCommonRythmGameEnd`: postfix 후킹 (`BgmMethodCallHooks.MethodCallPostfixVoid`)
     - 재생 시간과 BGM 길이 비교 로깅
   - `coMonitorGameEnd`: prefix/postfix 후킹 (`BgmGameEndMonitor.MonitorGameEndPrefix (구현: BgmGameEndMonitor.Adjustments.cs)/Postfix`)
     - Prefix: `BgmGameEndMonitor.AdjustMusicDataForBgmLength() (구현: BgmGameEndMonitor.Adjustments.cs)`로 4개 필드 조정
     - Postfix: `mKudosBoostEndSample` 필드도 업데이트
     - **현재 테스트 모드로 실행 허용됨** (return true)
   - `coClearRythmGameEnd`: prefix 후킹 (`BgmGameEndMonitor.ClearGameEndPrefix`)
     - 로그 제거됨

#### 제거된 진단 훅

`BgaVideoHooks`와 `HarmonyHookManager`는 2026-05-12 정리에서 제거되었습니다. 현재 BGA/BGM 처리는 `BgmBgaInjector`, `BgaInjector`, `BgaBgmSyncManager`, `BgmInjectorHooks` 계열이 담당합니다.

```
     └─ connectNodeDataArray 확인하여 홀드/페어리 노트 연결
```

## 알려진 제한사항 및 추정

### 1. 마지막 노트 플래그 설정 ✅

**현재 상태:**
- ✅ 모드는 변환 과정에서 자동으로 마지막 노트를 찾아 `isLast` 플래그를 `true`로 설정
- ✅ `connectNodeDataArray`의 끝 노트도 고려하여 정확한 마지막 노트 식별

**구현 내용:**
- `BmsNoteConverter.SetLastNoteFlag() (구현: BmsNoteConverter.Validation.cs)` 메서드에서 모든 노트와 연결 노트의 `perfectSample`을 비교
- 가장 큰 `perfectSample`을 가진 노트에 `isLast = true` 설정
- 게임이 마지막 노트를 올바르게 인식할 수 있도록 보장

### 2. 게임 종료 시간 조정 ✅

**현재 상태:**
- ✅ 모드는 `BgmGameEndMonitor`를 통해 게임 종료 시간을 자동으로 조정
- ✅ **⚠️ 핵심: 반드시 4개 필드를 조정해야 함** - 이 4개를 조정하지 않으면 게임이 조기에 종료됨
- ✅ `mRythmGameMusicData`의 4개 필드를 BGM 길이에 맞춰 조정:
  1. **`musicFadeOutEndSample`**: BGM 길이 (샘플) - 예: 158.016초 = 7,584,768 샘플 (48000Hz 기준)
  2. **`musicFadeOutStartSample`**: BGM 길이 - 1초 (샘플) - 예: 157.016초 = 7,536,768 샘플 (페이드 아웃 1초 전 시작)
  3. **`screenFadeOutStartSample`**: BGM 길이 - 1.5초 (샘플) - 예: 156.516초 = 7,512,768 샘플 (화면 페이드 아웃 1.5초 전 시작)
  4. **`screenFadeOutEndSample`**: BGM 길이 (샘플) - 예: 158.016초 = 7,584,768 샘플 (화면 페이드 아웃 종료)
- ✅ **⚠️ 중요**: 
  - 이 4개 필드를 모두 조정하지 않으면 게임이 원본 BGM 길이(약 120초)에 맞춰 조기에 종료됨. 반드시 4개 모두 조정해야 함!
  - **재시작 시에도 필수**: 게임을 재시작하면 플레이 씬이 다시 로드되므로, 재시작 시에도 자동으로 4개 필드가 조정됨
  - 다른 필드는 조정하지 않아도 됨
- ✅ 플레이 씬 로드 시 자동 조정 (`BgmGameEndMonitor.AdjustMusicDataOnSceneLoad() (구현: BgmGameEndMonitor.Adjustments.cs)`)
- ✅ `coMonitorGameEnd` 코루틴 후킹을 통한 주기적 조정 (현재는 테스트 모드로 실행 허용됨)

**구현 내용:**
- `BgmGameEndMonitor.AdjustMusicDataForBgmLength() (구현: BgmGameEndMonitor.Adjustments.cs)`: 4개 필드를 BGM 길이에 맞춰 조정  
  (현재 구현 위치: `BgmGameEndMonitor.Adjustments.cs`, 클래스는 partial 분리)
  - `BgmFinishTimeManager.GetTargetFinishTime()`로 목표 시간 가져오기
  - 없으면 `cBGMBeatManager.getAudioClip()`으로 직접 확인
  - `mRythmGameMusicData` 내부의 **반드시 조정해야 하는 4개 필드** (`AdjustMusicDataForBgmLength()`에서 조정) - **정확히 이 4개만 조정해야 함**:
    1. **`musicFadeOutEndSample`**: BGM 길이 (샘플) - 예: 158.016초 = 7,584,768 샘플 (48000Hz 기준)
    2. **`musicFadeOutStartSample`**: BGM 길이 - 1초 (샘플) - 예: 157.016초 = 7,536,768 샘플 (페이드 아웃 1초 전 시작)
    3. **`screenFadeOutStartSample`**: BGM 길이 - 1.5초 (샘플) - 예: 156.516초 = 7,512,768 샘플 (화면 페이드 아웃 1.5초 전 시작)
    4. **`screenFadeOutEndSample`**: BGM 길이 (샘플) - 예: 158.016초 = 7,584,768 샘플 (화면 페이드 아웃 종료)
- `BgmGameEndMonitor.AdjustMusicDataOnSceneLoad() (구현: BgmGameEndMonitor.Adjustments.cs)`: 플레이 씬 로드 시 자동 조정 (구현 위치: `BgmGameEndMonitor.Adjustments.cs`)
- `BgmGameEndMonitor.MonitorGameEndPrefix (구현: BgmGameEndMonitor.Adjustments.cs)/Postfix()`: `coMonitorGameEnd` 후킹 (구현 위치: `BgmGameEndMonitor.Adjustments.cs`, 현재 테스트 모드로 실행 허용)
  - Postfix에서 `mKudosBoostEndSample` 필드도 업데이트
- `BgmFinishTimeManager.SetFinishTime()`: BGM 길이와 마지막 노트 시간 중 더 큰 값으로 설정
- `BgmFinishTimeManager.GetTargetFinishTime()`: 설정된 목표 종료 시간 반환
- `BgmMonitorCoroutine.MonitorBgmFinishCoroutine()`: BGM 종료 시 결과 화면으로 전환
  - `BgmMethodCallHooks.HandleImportantMethodCall()`에서 `requestPlayAudio` 호출 시 자동 시작
  - 0.1초마다 `getCurrentSample()`로 재생 시간 확인
  - BGM 길이 도달 시 `requestCommonRythmGameEnd()` 호출

**추정:**
- 게임이 `mFairyNoteCreateDataArray`를 자동으로 분석하여 `mMaxTiming`을 계산할 가능성이 높음
- BGM 길이는 `cBGMBeatManager.setClip()` 호출 시 자동으로 설정될 가능성이 높음
- **하지만 실제 게임 종료는 `mRythmGameMusicData`의 4개 필드를 사용하는 것으로 추정됨**
  - 이 4개 필드가 게임 종료 시간을 결정하는 핵심 필드
  - 원본 게임은 이 필드들을 원본 BGM 길이(약 120초)에 맞춰 설정
  - 모드는 주입된 BGM 길이에 맞춰 이 4개 필드를 조정하여 게임이 정확히 종료되도록 함
  - **⚠️ 중요**: 
    - 이 4개 필드를 모두 조정하지 않으면 게임이 원본 BGM 길이(약 120초)에 맞춰 조기에 종료됨. 반드시 4개 모두 조정해야 함!
    - **재시작 시에도 필수**: 게임을 재시작하면 플레이 씬이 다시 로드되므로, 재시작 시에도 자동으로 4개 필드가 조정됨
    - 다른 필드는 조정하지 않아도 됨

**해결 완료:**
- ✅ BMS 차트가 원본보다 긴 경우: 4개 필드를 조정하여 게임이 정확히 종료됨
- ✅ BGM 길이 불일치: BGM 길이에 맞춰 4개 필드를 조정하여 게임이 정확히 종료됨
- ✅ BGM 모니터링 코루틴: BGM 종료 시 강제로 결과 화면으로 전환

### 3. BMS 작성 주의사항 ⚠️

**중요: 홀드/페어리 끝 노트 필수 배치**

홀드 노트(02) 또는 페어리 노트(11-18)를 사용하는 경우, 반드시 해당 끝 노트를 배치해야 합니다.

**주입 거부 정책:**
- 홀드/페어리 끝 노트를 배치하지 않으면 **BMS 노트 주입이 거부됩니다**.
- 로그에 "BMS 파일을 다시 확인해보세요!" 메시지가 출력됩니다.
- 게임은 원본 차트로 플레이됩니다.

**문제 발생 시나리오:**
- 페어리/홀드 끝 노트를 배치하지 않으면 노트가 제대로 처리되지 않습니다.
- 페어리/홀드 시작 노트만 있고 끝 노트가 없으면, 노트가 메인 배열에 남아있게 되어 성능 문제가 발생할 수 있습니다.
- 따라서 모드는 안전을 위해 주입을 거부합니다.

**올바른 BMS 작성 방법:**

**페어리 노트:**
```
#00111: 0011000000000000  // measure 1, 채널 11, 페어리 시작(11) ✅
#00211: 001A000000000000  // measure 2, 채널 11, 페어리 끝(1A) ✅ 필수!
```

**홀드 노트:**
```
#00111: 0002000000000000  // measure 1, 채널 11, 홀드 시작(02) ✅
#00211: 0019000000000000  // measure 2, 채널 11, 홀드 끝(19) ✅ 필수!
```

### 5. 노트 판정 시스템
