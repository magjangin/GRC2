# 게임 종료 시간 조정 가이드

이 문서는 주입된 BGM 길이에 맞춰 게임 종료 시간을 조정하는 방법과 관련 문제 해결 방법을 설명합니다.

**중요(현재 코드 기준)**:
- “종료 시간 조정 로직(필드 조정)” 자체는 후킹/씬 로드 흐름에서 호출될 수 있어 **커스텀 차트 선택 여부로 완전히 제한되어 있지 않습니다.**
- 다만, **실제로 문제를 해결해야 하는 상황은 ‘커스텀 BGM이 주입되어 원본 길이와 달라졌을 때’**이며,
  - `BgmFinishTimeManager`에 목표 시간이 설정되거나(주입 BGM 길이/마지막 노트 기반),
  - 주입된 BGM이 감지되어 모니터링이 시작될 때
  효과가 의미 있게 나타납니다.

## 개요

원본 게임은 고정된 BGM 길이(약 120초)에 맞춰 게임이 종료되도록 설계되어 있습니다. 하지만 **커스텀 차트 선택 시** 커스텀 BGM을 주입할 경우, BGM 길이가 다를 수 있으므로 게임 종료 시간을 조정해야 합니다.

**현재 코드의 동작 관점에서 정리하면:**
- **커스텀 BGM 주입**: `BgmBgaInjector` 쪽이 `CustomAssetManager.IsCustomChartSelected()` 상태에서만 주입 루프를 진행합니다.
- **목표 종료 시간 계산**: 주입된 BGM 길이와 마지막 노트(`perfectSample`)를 비교해 큰 값으로 설정합니다. (주입 BGM이 없으면 목표 시간이 0일 수 있음)
- **4개 필드 조정**: 목표 시간이 있으면 그 값을 기준으로 조정되며, 목표 시간이 없을 때는 `cBGMBeatManager.getAudioClip().length`에서 길이를 읽어 조정할 수 있습니다(즉 “커스텀 선택”만으로 제한되지는 않음).

## 핵심: 4개 필드 조정

**⚠️ 중요**: 게임이 정확히 종료되려면 반드시 **4개 필드를 모두 조정**해야 합니다. 하나라도 빠지면 게임이 원본 BGM 길이(약 120초)에 맞춰 조기에 종료됩니다.

### 조정해야 하는 4개 필드

`mRythmGameMusicData` 객체 내부의 다음 4개 필드를 조정합니다:

1. **`musicFadeOutEndSample`**: BGM 길이 (샘플)
   - 예: 158.016초 = 7,584,768 샘플 (48000Hz 기준)
   - 음악 페이드 아웃 종료 시점

2. **`musicFadeOutStartSample`**: BGM 길이 - 1초 (샘플)
   - 예: 157.016초 = 7,536,768 샘플
   - 음악 페이드 아웃 시작 시점 (1초 전)

3. **`screenFadeOutStartSample`**: BGM 길이 - 1.5초 (샘플)
   - 예: 156.516초 = 7,512,768 샘플
   - 화면 페이드 아웃 시작 시점 (1.5초 전)

4. **`screenFadeOutEndSample`**: BGM 길이 (샘플)
   - 예: 158.016초 = 7,584,768 샘플
   - 화면 페이드 아웃 종료 시점

### 샘플 변환 공식

```
샘플 = 초 × 48000
```

예시:
- BGM 길이: 158.016초
- 샘플: 158.016 × 48000 = 7,584,768 샘플

## 구현

### 조정 메서드 (`BgmGameEndMonitor.AdjustMusicDataForBgmLength (구현: BgmGameEndMonitor.Adjustments.cs)`)

```csharp
public static void AdjustMusicDataForBgmLength()
{
    // BGM 길이 가져오기
    float bgmLength = BgmFinishTimeManager.GetTargetFinishTime();
    if (bgmLength <= 0)
    {
        // 대체 방법: cBGMBeatManager에서 직접 가져오기
        var bgmManager = BgmInjector.GetBgmManagerInstance();
        if (bgmManager != null)
        {
            var audioClip = BgmInjector.GetAudioClip(bgmManager);
            if (audioClip != null)
            {
                bgmLength = audioClip.length;
            }
        }
    }

    if (bgmLength <= 0) return;

    // 샘플 변환
    int bgmLengthSample = (int)(bgmLength * 48000);
    int fadeOutStartSample = (int)((bgmLength - 1.0f) * 48000);
    int screenFadeOutStartSample = (int)((bgmLength - 1.5f) * 48000);

    // mRythmGameMusicData 찾기
    var musicData = FindMusicData();
    if (musicData == null) return;

    // 4개 필드 조정
    SetFieldValue(musicData, "musicFadeOutEndSample", bgmLengthSample);
    SetFieldValue(musicData, "musicFadeOutStartSample", fadeOutStartSample);
    SetFieldValue(musicData, "screenFadeOutStartSample", screenFadeOutStartSample);
    SetFieldValue(musicData, "screenFadeOutEndSample", bgmLengthSample);
}
```

## 조정 시점

### 1. 플레이 씬 로드 시 (`BgmGameEndMonitor.AdjustMusicDataOnSceneLoad (구현: BgmGameEndMonitor.Adjustments.cs)`)

```csharp
public static void AdjustMusicDataOnSceneLoad()
{
    // 플레이 씬 로드 시 자동으로 4개 필드 조정
    AdjustMusicDataForBgmLength();
}
```

**호출 위치:**
- `SceneDetector.OnSceneWasLoaded()`에서 `FairyModeScene` 또는 `PlayMovieScene` 로드 시 호출

### 2. 게임 진행 중 (`BgmGameEndMonitor.MonitorGameEndPrefix (구현: BgmGameEndMonitor.Adjustments.cs)/Postfix`)

```csharp
public static bool MonitorGameEndPrefix()
{
    // coMonitorGameEnd 코루틴 후킹
    // Prefix에서 4개 필드 조정
    AdjustMusicDataForBgmLength();
    return true; // 원본 메서드 실행 허용
}
```

**호출 위치:**
- `cRythmGameManager.coMonitorGameEnd` 코루틴을 후킹하여 주기적으로 조정
- 현재는 테스트 모드로 실행 허용됨

## 추가 보호 메커니즘

### BGM 모니터링 코루틴 (`BgmMonitorCoroutine`)

```csharp
private static IEnumerator MonitorBgmFinishCoroutine()
{
    while (true)
    {
        yield return new WaitForSeconds(0.1f);
        
        var bgmManager = BgmInjector.GetBgmManagerInstance();
        if (bgmManager != null)
        {
            int currentSample = BgmInjector.GetCurrentSample(bgmManager);
            float bgmLength = BgmFinishTimeManager.GetTargetFinishTime();
            
            if (bgmLength > 0 && currentSample >= bgmLength * 48000)
            {
                // BGM 종료 시 결과 화면으로 전환
                BgmInjector.RequestGameEnd();
                yield break;
            }
        }
    }
}
```

**동작:**
- 0.1초마다 BGM 재생 시간 확인
- BGM 길이 도달 시 `requestCommonRythmGameEnd()` 호출하여 결과 화면으로 전환

## 문제 해결

### 문제 1: 게임이 조기에 종료됨

**증상:**
- 게임이 원본 BGM 길이(약 120초)에 맞춰 조기에 종료됨
- 주입된 BGM이 끝나기 전에 게임이 종료됨

**원인:**
- 4개 필드 중 일부만 조정됨
- 조정 시점이 너무 늦음
- `mRythmGameMusicData` 객체를 찾지 못함

**해결 방법:**
1. **4개 필드 모두 조정 확인**: 로그에서 4개 필드가 모두 조정되었는지 확인
2. **조정 시점 확인**: 플레이 씬 로드 시 조정이 실행되는지 확인
3. **객체 찾기 확인**: `mRythmGameMusicData` 객체를 정확히 찾는지 확인

### 문제 2: 게임이 너무 늦게 종료됨

**증상:**
- 게임이 BGM이 끝난 후에도 계속 진행됨
- 결과 화면으로 전환되지 않음

**원인:**
- BGM 모니터링 코루틴이 시작되지 않음
- `requestCommonRythmGameEnd()` 호출이 실패함

**해결 방법:**
1. **모니터링 코루틴 확인**: `BgmMethodCallHooks.HandleImportantMethodCall()`에서 `requestPlayAudio` 호출 시 자동 시작되는지 확인
2. **BGM 길이 확인**: `BgmFinishTimeManager.GetTargetFinishTime()`이 정확한 값을 반환하는지 확인
3. **로그 확인**: `[BgmMonitorCoroutine]` 로그 메시지 확인

### 문제 3: 필드 조정이 반영되지 않음

**증상:**
- 필드를 조정했지만 게임이 여전히 원본 시간에 종료됨

**원인:**
- 잘못된 객체에 필드 설정
- 필드 이름 오타
- Reflection 캐싱 문제

**해결 방법:**
1. **필드 이름 확인**: 정확한 필드 이름 사용 (`musicFadeOutEndSample` 등)
2. **객체 확인**: `mRythmGameMusicData` 객체가 정확히 찾아지는지 확인
3. **캐시 초기화**: Reflection 캐시를 초기화하고 다시 시도
