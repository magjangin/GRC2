# 리플렉션 및 필드 접근 시스템

## 개요

이 문서는 GRC2 모드에서 사용하는 리플렉션(Reflection) 시스템과 필드 접근 최적화 기법을 상세히 설명합니다. 리플렉션은 런타임에 타입 정보를 조회하고 조작할 수 있는 강력한 기능이지만, 성능 오버헤드가 크기 때문에 캐싱 전략이 필수적입니다.

## 리플렉션 시스템 아키텍처

### 전체 구조

```
ReflectionHelper (타입/메서드 탐색)
    ├─ 타입 캐시 (_typeCache)
    ├─ 메서드 캐시 (_methodCache)
    └─ 어셈블리 캐시 (_cachedAssemblies)

FieldAccessHelper (필드 접근)
    └─ 필드 캐시 (_fieldCache)

GameTypeLoader (게임 타입 로딩)
    └─ 게임 타입 저장 (NoteCreateDataType, Enum 타입들)
```

### 핵심 원칙

1. **한 번 찾고, 여러 번 사용**: 타입/메서드/필드 정보를 캐시에 저장
2. **Dictionary 기반 O(1) 검색**: 문자열 키로 빠른 조회
3. **Null 캐싱**: 찾지 못한 항목도 캐시하여 반복 검색 방지
4. **지연 초기화**: 필요할 때만 어셈블리 배열 생성

## ReflectionHelper 클래스

### 클래스 구조

```csharp
public static class ReflectionHelper
{
    // 타입 캐시: "Namespace.ClassName" → Type
    private static Dictionary<string, Type> _typeCache = new Dictionary<string, Type>();
    
    // 메서드 캐시: "TypeName.MethodName" → MethodInfo
    private static Dictionary<string, MethodInfo> _methodCache = new Dictionary<string, MethodInfo>();
    
    // 어셈블리 배열 캐시
    private static Assembly[] _cachedAssemblies = null;
}
```

### FindType 메서드

**목적**: 타입 이름으로 Type 객체 찾기

```csharp
public static Type FindType(string typeName)
{
    // ⚡ 1단계: 캐시에서 먼저 검색
    if (_typeCache.TryGetValue(typeName, out var cachedType))
    {
        return cachedType; // O(1) 반환
    }

    // 2단계: 모든 어셈블리에서 검색
    var assemblies = GetCachedAssemblies();
    foreach (var assembly in assemblies)
    {
        try
        {
            var type = assembly.GetType(typeName);
            if (type != null)
            {
                // ⚡ 캐시에 저장
                _typeCache[typeName] = type;
                return type;
            }
        }
        catch { }
    }
    
    // ⚡ 3단계: null도 캐시 (다음 호출 시 빠르게 반환)
    _typeCache[typeName] = null;
    return null;
}
```

**성능 분석:**

| 호출 | 캐시 상태 | 검색 시간 | 설명 |
|------|-----------|-----------|------|
| 1차 | 없음 | ~10ms | 모든 어셈블리 검색 |
| 2차 | 있음 | ~0.001ms | Dictionary 조회 (O(1)) |
| 3차 | 있음 | ~0.001ms | Dictionary 조회 (O(1)) |

**사용 예제:**

```csharp
// 첫 호출: 느림 (어셈블리 검색)
Type noteType = ReflectionHelper.FindType("IntiCreates.NoteCreateData");

// 이후 호출: 빠름 (캐시 사용)
Type noteType2 = ReflectionHelper.FindType("IntiCreates.NoteCreateData");
Type noteType3 = ReflectionHelper.FindType("IntiCreates.NoteCreateData");
```

### FindMethod 메서드

**목적**: 타입과 메서드 이름으로 MethodInfo 찾기

```csharp
public static MethodInfo FindMethod(string typeName, string methodName, 
    BindingFlags flags = BindingFlags.Public | BindingFlags.Instance | BindingFlags.NonPublic, 
    bool silent = false)
{
    var cacheKey = $"{typeName}.{methodName}";
    
    // ⚡ 1단계: 캐시에서 먼저 검색
    if (_methodCache.TryGetValue(cacheKey, out var cachedMethod))
    {
        return cachedMethod;
    }

    // 2단계: 어셈블리에서 타입 찾기
    var assemblies = GetCachedAssemblies();
    foreach (var assembly in assemblies)
    {
        try
        {
            var type = assembly.GetType(typeName);
            if (type != null)
            {
                var method = type.GetMethod(methodName, flags);
                if (method != null)
                {
                    // ⚡ 캐시에 저장
                    _methodCache[cacheKey] = method;
                    
                    if (!silent)
                    {
                        MelonLogger.Msg($"[ReflectionHelper] 메서드 발견: {typeName}.{methodName}");
                    }
                    return method;
                }
            }
        }
        catch { }
    }
    
    // ⚡ 3단계: null도 캐시
    _methodCache[cacheKey] = null;
    
    if (!silent)
    {
        MelonLogger.Msg($"[ReflectionHelper] 메서드를 찾을 수 없습니다: {typeName}.{methodName}");
    }
    return null;
}
```

**사용 예제:**

```csharp
// BGM 관련 메서드 찾기
MethodInfo setClipMethod = ReflectionHelper.FindMethod(
    "IntiCreates.cBGMBeatManager", 
    "setClip",
    BindingFlags.Public | BindingFlags.Instance);

if (setClipMethod != null)
{
    // 메서드 호출
    setClipMethod.Invoke(bgmManager, new object[] { audioClip });
}
```

### GetCachedAssemblies 메서드

**목적**: 어셈블리 배열 캐싱으로 반복 할당 방지

```csharp
private static Assembly[] GetCachedAssemblies()
{
    if (_cachedAssemblies == null)
    {
        _cachedAssemblies = AppDomain.CurrentDomain.GetAssemblies();
    }
    return _cachedAssemblies;
}
```

**최적화 효과:**

```csharp
// ❌ 나쁜 예: 매번 배열 할당
var assemblies = AppDomain.CurrentDomain.GetAssemblies(); // 매번 새 배열 생성

// ✅ 좋은 예: 캐시된 배열 사용
var assemblies = GetCachedAssemblies(); // 한 번만 생성
```

### ClearCache 메서드

**목적**: 메모리 정리 또는 어셈블리 재로드 시 캐시 초기화

## FieldAccessHelper 클래스

```csharp
public static class FieldAccessHelper
{
    // 필드 정보 캐시: "fieldName" → FieldInfo

    public const string FIELD_DIRECTION_INDEX = "directionIndex";

    public const string FIELD_NOTE_SIZE = "noteSize";
    public const string FIELD_CONNECT_NODE_DATA_ARRAY = "connectNodeDataArray";
    public const string FIELD_IS_CRITICAL = "isCritical";
    public const string FIELD_IS_LAST = "isLast";
    // ... 기타 필드들
}
```

### GetCachedField 메서드

**목적**: 필드 정보 캐싱으로 반복적인 GetField 호출 최소화

```csharp
public static FieldInfo GetCachedField(string fieldName)
{
    if (GameTypeLoader.NoteCreateDataType == null)
    {
        return null;
    }

    // ⚡ TryGetValue를 사용하여 ContainsKey + [] 패턴 최적화
    if (!_fieldCache.TryGetValue(fieldName, out var cachedField))
    {
        // 캐시에 없으면 검색
        var field = GameTypeLoader.NoteCreateDataType.GetField(fieldName,
            BindingFlags.Public | BindingFlags.NonPublic | BindingFlags.Instance);
        
        // 캐시에 저장 (null도 저장)
        _fieldCache[fieldName] = field;
        return field;
    }

    return cachedField;
}
```

**성능 비교:**

```csharp
// ❌ 나쁜 예: 매번 GetField 호출 (느림)
for (int i = 0; i < 1000; i++)
{
    var field = noteType.GetField("perfectSample", BindingFlags.Public | BindingFlags.Instance);
    field.SetValue(note, value);
}
// 실행 시간: ~50ms

// ✅ 좋은 예: 캐시된 FieldInfo 사용 (빠름)
for (int i = 0; i < 1000; i++)
{
    var field = FieldAccessHelper.GetCachedField("perfectSample");
    field.SetValue(note, value);
}
// 실행 시간: ~5ms (10배 빠름)
```

### SetFieldValue 메서드

**목적**: 필드 값 설정 (캐시 사용)


## 캐싱 전략

### 1. Dictionary 기반 캐싱

**장점:**
- O(1) 검색 시간
- 문자열 키로 직관적 접근
- 메모리 효율적

**구현:**

```csharp
// 타입 캐시
private static Dictionary<string, Type> _typeCache = new Dictionary<string, Type>();

// 사용
if (_typeCache.TryGetValue(typeName, out var type))
{
    return type; // 캐시 히트
}
```

### 2. Null 캐싱

**목적**: 찾지 못한 항목도 캐시하여 반복 검색 방지

```csharp
// 타입을 찾지 못한 경우
if (type == null)
{
    _typeCache[typeName] = null; // null도 캐시
    return null;
}
```

**효과:**

```csharp
// 첫 호출: 모든 어셈블리 검색 (느림)
var type1 = ReflectionHelper.FindType("NonExistentType"); // null

// 두 번째 호출: 캐시에서 null 반환 (빠름)
var type2 = ReflectionHelper.FindType("NonExistentType"); // null (즉시 반환)
```

### 3. 지연 초기화

**목적**: 필요할 때만 리소스 할당

```csharp
private static Assembly[] _cachedAssemblies = null;

private static Assembly[] GetCachedAssemblies()
{
    if (_cachedAssemblies == null) // 첫 호출 시에만 초기화
    {
        _cachedAssemblies = AppDomain.CurrentDomain.GetAssemblies();
    }
    return _cachedAssemblies;
}
```

### 4. 필드 이름 상수화

**목적**: 오타 방지 및 리팩토링 용이성

```csharp
// ❌ 나쁜 예: 문자열 리터럴 (오타 위험)
FieldAccessHelper.SetFieldValue(obj, "perfectSample", value);
FieldAccessHelper.SetFieldValue(obj, "perfectSampl", value); // 오타!

// ✅ 좋은 예: 상수 사용 (컴파일 타임 체크)
FieldAccessHelper.SetFieldValue(obj, FieldAccessHelper.FIELD_PERFECT_SAMPLE, value);
```

## 성능 최적화 기법

### 1. TryGetValue 패턴

**목적**: ContainsKey + [] 패턴보다 효율적

```csharp
// ❌ 나쁜 예: 두 번 검색
if (_cache.ContainsKey(key))
{
    return _cache[key]; // Dictionary를 두 번 검색
}

// ✅ 좋은 예: 한 번만 검색
if (_cache.TryGetValue(key, out var value))
{
    return value; // Dictionary를 한 번만 검색
}
```

### 2. 조기 반환 (Early Return)

**목적**: 불필요한 작업 방지

```csharp
public static FieldInfo GetCachedField(string fieldName)
{
    // 조기 반환: 타입이 없으면 즉시 종료
    if (GameTypeLoader.NoteCreateDataType == null)
    {
        return null;
    }

    // 캐시 검색
    if (!_fieldCache.TryGetValue(fieldName, out var cachedField))
    {
        // ...
    }

    return cachedField;
}
```

### 3. 예외 처리 최소화

**목적**: try-catch 오버헤드 감소

```csharp
// ✅ 좋은 예: 필요한 곳에만 try-catch
public static void SetFieldValue(object obj, string fieldName, object value)
{
    try
    {
        var field = GetCachedField(fieldName); // 캐시 조회는 안전
        if (field != null)
        {
            field.SetValue(obj, value); // SetValue만 예외 가능
        }
    }
    catch (Exception ex)
    {
        MelonLogger.Warning($"필드 설정 실패: {ex.Message}");
    }
}
```

### 4. 배열 할당 최소화

**목적**: GC 압력 감소

```csharp
// ❌ 나쁜 예: 매번 배열 할당
for (int i = 0; i < 1000; i++)
{
    var assemblies = AppDomain.CurrentDomain.GetAssemblies(); // 1000번 할당
}

// ✅ 좋은 예: 한 번만 할당
var assemblies = GetCachedAssemblies(); // 1번 할당
for (int i = 0; i < 1000; i++)
{
    // assemblies 사용
}
```

### 1. 항상 캐시 사용

```csharp
// ✅ 좋은 예
var field = FieldAccessHelper.GetCachedField("perfectSample");

// ❌ 나쁜 예

```

### 2. 필드 이름 상수 사용

```csharp
// ✅ 좋은 예
FieldAccessHelper.SetFieldValue(obj, FieldAccessHelper.FIELD_PERFECT_SAMPLE, value);

// ❌ 나쁜 예
FieldAccessHelper.SetFieldValue(obj, "perfectSample", value);
```

### 3. Null 체크

```csharp
// ✅ 좋은 예
var field = FieldAccessHelper.GetCachedField("perfectSample");
if (field != null)
{
    field.SetValue(obj, value);
}

// ❌ 나쁜 예
var field = FieldAccessHelper.GetCachedField("perfectSample");
field.SetValue(obj, value); // NullReferenceException 위험
```
